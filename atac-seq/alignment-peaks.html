QC 五指標

ENCODE 5 個核心 QC

📈

TSS Enrichment

TSS ±2 kb 內訊號 / 背景訊號的比值。ATAC 最重要的單一指標。> 7 高、5–7 可、< 5 低。

公式： TSS_e = mean(coverage, TSS ± 100 bp) / mean(coverage, TSS ± 1.9–2 kb)

Signal at TSS ±2 kb / background signal. Single most important ATAC metric. > 7 high, 5–7 acceptable, < 5 low.

Formula: TSS_e = mean(coverage, TSS ± 100 bp) / mean(coverage, TSS ± 1.9–2 kb)

🎯

FRiP (Fraction of Reads in Peaks)

落在 peaks 內的 read 比例。ENCODE 建議 > 0.3；> 0.5 優秀。

公式： FRiP = reads_in_peaks / total_reads

Read fraction in peaks. ENCODE recommends > 0.3; > 0.5 is excellent.

Formula: FRiP = reads_in_peaks / total_reads

🔄

PBC1 (PCR Bottleneck Coefficient)

Library 複雜度指標。> 0.7 良好。過低代表 PCR 過頭或樣本不足。

公式： PBC1 = N₁ / N_d，N₁ = 只出現 1 次的 fragment 位置數；N_d = 不同 fragment 位置總數。

Library complexity. > 0.7 good; low → PCR over-amplification.

Formula: PBC1 = N₁ / N_d, where N₁ = # positions appearing exactly once, N_d = # distinct positions.

🧪

NRF (Non-Redundant Fraction)

Non-duplicate reads / total reads。ENCODE 建議 > 0.8。

公式： NRF = N_d / N_r，N_r = 總 mapped read 數。

Non-duplicate reads / total reads. ENCODE recommends > 0.8.

Formula: NRF = N_d / N_r, N_r = total mapped reads.

🔢

Peak 數量

典型人類樣本：50,000 – 150,000 peaks (q < 0.05)。過少代表 under-tagmented 或 chrM% 高；過多代表噪音過頭。

Typical human samples: 50,000 – 150,000 peaks (q < 0.05). Too few → under-tagmented or high chrM%; too many → noise-driven.

🪞

Cross-Replicate IDR

Irreproducibility Discovery Rate。兩個 replicates 的 peak rank 一致性。ENCODE 標準：IDR < 0.05 視為 reproducible peak。

Irreproducibility Discovery Rate. Consistency of peak ranks between replicates. ENCODE standard: IDR < 0.05 = reproducible peak.

QC 速查表 TSS enrichment: > 7 (high), 5–7 (med), < 5 (fail) FRiP: > 0.3 (acceptable), > 0.5 (excellent) PBC1: > 0.7, NRF: > 0.8 chrM%: < 20% (post-removal), peak count: 50k–150k (human) 所有指標都應達標

程式碼

完整流程程式碼

# MACS3 推薦 ATAC paired-end 設定（ENCODE 規格）
# --nomodel: 跳過建模（ATAC 不像 ChIP 有單一 fragment size）
# --shift -75 --extsize 150: 每個 Tn5 切位中心畫 150 bp 視窗
macs3 callpeak \
  -f BAMPE -t shifted.sorted.bam \
  -g hs -n sample1 \
  --nomodel --shift -75 --extsize 150 \
  -q 0.05 --keep-dup all \
  --call-summits \
  --outdir macs3_out/

# 輸出: sample1_peaks.narrowPeak (9 cols), sample1_peaks.xls, sample1_summits.bed

# 想要 broad peaks（適合廣域組蛋白修飾如 H3K27me3）：
macs3 callpeak ... --broad --broad-cutoff 0.1
# Genrich 是 ATAC 友好：內建 -j（ATAC mode）、-r（替代 PCR-dup）、-e（排除 chrM）
samtools sort -n shifted.sorted.bam -o sorted_qname.bam   # Genrich 要 qname-sorted

Genrich -t sorted_qname.bam \
  -j -y -r -e chrM \
  -q 0.05 \
  -o sample1.narrowPeak \
  -k sample1.pileup.bedgraph \
  -f sample1.log

# 多 replicates 聯合呼叫 (Genrich 內建支援):
Genrich -t rep1.bam,rep2.bam,rep3.bam \
  -j -y -r -e chrM -o joint.narrowPeak
# 1. FRiP — 落在 peaks 的 read 比例
in_peak=$(bedtools intersect -u -a shifted.sorted.bam -b sample1_peaks.narrowPeak | samtools view -c -)
total=$(samtools view -c shifted.sorted.bam)
echo "FRiP = $(echo "scale=3; $in_peak / $total" | bc)"

# 2. TSS enrichment (deepTools) — 視覺化 + 計算
computeMatrix reference-point \
  -S sample1.bw -R hg38_tss.bed \
  --referencePoint TSS -a 2000 -b 2000 \
  --skipZeros -p 8 \
  --outFileName tss_matrix.gz

plotProfile -m tss_matrix.gz -o tss_profile.png \
  --regionsLabel "TSS" --plotTitle "TSS Enrichment" \
  --yAxisLabel "signal"

# 3. PBC1 / NRF (encode-tools)
python -c "
import pysam
positions = {}
for r in pysam.AlignmentFile('shifted.sorted.bam','rb'):
  if r.is_read1 and r.is_proper_pair:
    key = (r.reference_name, r.reference_start, r.template_length)
    positions[key] = positions.get(key,0) + 1
N1 = sum(1 for v in positions.values() if v==1)
Nd = len(positions); Nr = sum(positions.values())
print(f'NRF = {Nd/Nr:.3f}  PBC1 = {N1/Nd:.3f}')
"
# ataqv (Parker lab) — ATAC 專用 QC 包，一次出 HTML 報表
ataqv --peak-file sample1_peaks.narrowPeak \
      --tss-file hg38_tss.bed \
      --metrics-file sample1.metrics.json.gz \
      --threads 8 \
      --name sample1 human shifted.sorted.bam

mkarv viewer/ *.metrics.json.gz   # 多樣本合併成單一 HTML viewer
# IDR — replicate 之間的 peak 一致性
# 先分別 call peaks（rep1, rep2, pooled）
idr --samples rep1.narrowPeak rep2.narrowPeak \
    --peak-list pooled.narrowPeak \
    --input-file-type narrowPeak \
    --rank p.value \
    --output-file idr.txt \
    --plot --idr-threshold 0.05

# 取 IDR < 0.05 的 peaks 為「reproducible」
awk '$11 <= 0.05' idr.txt > idr_passed.narrowPeak
差異開放區

差異開放區 (DAR)

類似 RNA-seq 的 DE，但作用在 peak count 上。兩大主流：

Like RNA-seq DE but on peak counts. Two mainstream approaches:

DiffBind
  • 基於 peak-level 計數
  • Peak-level counts
  • 後端：DESeq2 / edgeR
  • Backend: DESeq2 / edgeR
  • 優：簡單、與 RNA-seq 一致；缺：依賴 peak 集合（union/consensus）
  • Pros: simple, RNA-like; Cons: peak-set dependent
VS
csaw
  • 基於 sliding window 計數
  • Sliding-window counts
  • 後端：edgeR (QL F-test)
  • Backend: edgeR (QL F-test)
  • 優：不依賴預先 peak set；缺：複雜、計算量大
  • Pros: peak-set free; Cons: complex, heavier compute

library(DiffBind)
# samples.csv 含 SampleID, Condition, Replicate, bamReads, Peaks, PeakCaller
samples <- dba(sampleSheet = "samples.csv")
samples <- dba.count(samples, summits = 200)            # 以 summit ±200 bp 為計數區
samples <- dba.normalize(samples, method = DBA_DESEQ2)
samples <- dba.contrast(samples, categories = DBA_CONDITION,
                          minMembers = 2)
samples <- dba.analyze(samples)
db <- dba.report(samples, th = 0.05, fold = 1)         # |log2FC|≥1, FDR≤0.05

dba.plotMA(samples)           # MA plot
dba.plotVolcano(samples)
dba.plotHeatmap(samples, contrast = 1, correlations = FALSE)
library(csaw)
# 1. sliding-window counts (width 150 bp, ext 200 bp)
counts <- windowCounts(bam.files, width = 150, ext = 200,
                       param = readParam(minq = 30, pe = "both"))
# 2. 過濾低訊號 window
counts <- filterWindowsGlobal(counts)
keep <- counts$filter$filter > log2(3)
counts <- counts[keep,]
# 3. 標準化（trended bias）
counts <- normOffsets(counts)
# 4. edgeR QL F-test
y <- asDGEList(counts); y <- estimateDisp(y, design)
fit <- glmQLFit(y, design, robust = TRUE)
res <- glmQLFTest(fit, contrast = c.matrix)
# 5. 合併鄰近 window 為 regions
merged <- mergeResults(counts, res$table, tol = 500)
library(DESeq2); library(GenomicAlignments)
# 1. 取所有樣本的 union peaks
union_peaks <- reduce(do.call(c, lapply(peak_files, rtracklayer::import)))
# 2. 算每個樣本在 union peak 的 count matrix
counts <- summarizeOverlaps(features = union_peaks, reads = bam_files,
                          mode = "Union", ignore.strand = TRUE)
# 3. DESeq2 標準流程
dds <- DESeqDataSet(counts, design = ~ condition)
dds <- DESeq(dds)
res <- results(dds, contrast = c("condition", "treated", "ctrl"),
               alpha = 0.05)
summary(res)
真實案例

真實案例：TCGA Pan-Cancer ATAC

🎗️ 癌症 enhancer 圖譜的誕生

Corces MR, Granja JM, …, Chang HY. Science 2018, 362:eaav1898. (NCI TCGA + Stanford)

規模：410 個 ATAC 樣本，跨 23 種 TCGA 癌症（BRCA、LUAD、COAD、GBM、PRAD…）。每個樣本用 omni-ATAC，平均 6800 萬 paired-end reads。

流程：Bowtie2 → MACS2 → 算 union peak set（562,709 個 reproducible peaks）→ 每樣本算 count matrix → DESeq2 找 cancer-specific peaks。

關鍵 QC 報表：

  • TSS enrichment 中位數 7.5（範圍 4.2–18.6）
  • FRiP 中位數 0.38
  • 每樣本 50k–180k peaks
  • 排除 TSS_e < 4 的樣本（剩 410）

關鍵生物學發現：

  1. ~30% 癌症 enhancer 在對應正常組織中不開放——是 tumour-specific。
  2. BRCA luminal subtype 在 FOXA1 / ESR1 motif 富集；basal subtype 在 SOX10 / TFAP2A。
  3. 新發現的 enhancer 與 GWAS noncoding variant 高度共定位——把 SNP 連到表觀調控機制。
  4. 釋出資料：tcga.atacseq.xenahubs.net，已被引用 > 1000 次。

學到什麼？大規模 ATAC 需要：(a) 統一 protocol（omni-ATAC）；(b) 統一 union peak set；(c) 嚴格 QC（TSS_e>5）；(d) batch effect 處理。這篇是「ATAC 做大規模臨床」的範本。

Scale: 410 ATAC samples across 23 TCGA cancers (BRCA, LUAD, COAD, GBM, PRAD…). Each sample run with omni-ATAC, ~68M paired-end reads on average.

Pipeline: Bowtie2 → MACS2 → 562,709-peak reproducible union set → per-sample counts → DESeq2 for cancer-specific peaks.

Key QC report:

  • Median TSS enrichment 7.5 (range 4.2–18.6)
  • Median FRiP 0.38
  • 50k–180k peaks per sample
  • Excluded samples with TSS_e < 4 (final n = 410)

Key biological findings:

  1. ~30% of cancer enhancers are not open in matched normal tissue — tumour-specific.
  2. BRCA luminal subtype enriches FOXA1 / ESR1 motifs; basal enriches SOX10 / TFAP2A.
  3. New enhancers co-localise with GWAS noncoding variants — linking SNPs to epigenetic mechanism.
  4. Data release: tcga.atacseq.xenahubs.net; cited > 1,000 times.

Lessons: Large-scale ATAC needs (a) unified protocol (omni-ATAC); (b) unified union peaks; (c) strict QC (TSS_e>5); (d) batch-effect control. The template for "ATAC at clinical scale."

陷阱

5 個常見錯誤

用 MACS 預設參數（ChIP 為目的設計）跑 ATAC：會建模 fragment size 為單峰（ATAC 是多峰），peaks 形狀變形。務必加 --nomodel --shift -75 --extsize 150。Using MACS defaults (ChIP-designed) for ATAC: tries to model a single fragment size (ATAC is multimodal), peaks deform. Always add --nomodel --shift -75 --extsize 150.
沒移除 chrM 就算 FRiP：chrM 是巨大 noise pool；FRiP 會被嚴重低估。先排除 chrM 再算。Computing FRiP without removing chrM: chrM is a giant noise pool; FRiP gets badly underestimated. Filter chrM first.
沒做 IDR 就比較 replicates：單一樣本的 peaks 可能有 FDR 5% 的噪音；replicates IDR < 0.05 才是真正穩定 peaks。Comparing replicates without IDR: a single sample's peaks include ~5% FDR noise; only replicate IDR < 0.05 marks robust peaks.
DiffBind 用不同 peak caller 結果混搭：MACS3 narrowPeak 與 Genrich narrowPeak 的 summit 定義不同；統一用同一 caller。Mixing peak callers in DiffBind: MACS3 and Genrich define summits differently. Stick to one caller.
沒去 blacklist 就 motif enrichment：ENCODE blacklist 含重複序列；motif 富集會被假陽性主導。Motif enrichment without removing blacklist: ENCODE blacklist holds repeat regions; enrichment gets dominated by artifacts.

對。你已用 picard MarkDuplicates 標記+移除，所以叫 peak 時告訴 MACS「不要再 dedup」（避免重複工作 + 防止 MACS 用自己較弱的 dedup）。Yes. You already MarkDuplicates'd, so tell MACS not to dedup again (avoid double work + MACS's weaker dedup logic).
高品質：> 0.7。中等：0.5–0.7。低於 0.5 表示 replicates 變異太大，建議檢查批次效應或 TSS_e 是否不一致。High quality: > 0.7. Medium: 0.5–0.7. Below 0.5 → too-large replicate variance; check batch effects or TSS_e disparity.
和 RNA-seq 不同：peaks 是 binary-ish（open 或 not），fold 通常較小。建議 |log2FC|≥1 (= 2-fold) + FDR≤0.05 起步，視專案調整。Unlike RNA-seq, peaks are near-binary; fold changes are smaller. Start with |log2FC|≥1 + FDR≤0.05, adjust by project.
不建議：兩者長度尺度差很大（narrow ~300 bp、broad 數 kb），counts 不可比。各自分開做。No: very different length scales (narrow ~300 bp, broad several kb); counts incomparable. Do them separately.
