Bulk vs Single-cell 整合

📦

Bulk 整合

同一批樣本同時做 RNA-seq + ATAC-seq → 在條件差異 (case vs control) 中比較 DEG 與 DAR 的重疊。常用 footprint 變化 + TF target 的 RNA 變化做 GRN 推論。

Same samples run for RNA + ATAC → compare DEG vs DAR overlap across conditions. GRN inference combines footprint changes with target-gene RNA changes.

🔬

scMultiome 整合

在「同一細胞」同時測 RNA + ATAC（10x Multiome）。每細胞同時有表達與開放性向量，可直接計算 peak-gene 相關。工具：Signac、ArchR、MUON。

RNA + ATAC in the SAME cell (10x Multiome). Every cell carries expression + accessibility vectors — direct peak-gene correlations. Tools: Signac, ArchR, MUON.

如何聯結 peak 與 gene？

典型方法有三種：

Three common approaches:

🌳 peak2gene strategies

距離：
最簡單。peak 對應最近 TSS 或 ±100 kb 內所有基因。優點：快、無需資料；缺點：忽略真實生物學。
相關性：
scMultiome：對每對 (peak, gene) 算跨細胞的相關係數。優：資料驅動；缺：需大量細胞。
3D 接觸：
結合 Hi-C / Capture-C / HiChIP 等 chromatin loop 資料；最生物學可信，但實驗成本最高。
Distance:
Simplest. Link peak to nearest TSS or all genes within ±100 kb. Fast; no extra data; ignores biology.
Correlation:
In scMultiome: per (peak, gene) pair, correlate across cells. Data-driven; needs many cells.
3D contact:
Use Hi-C / Capture-C / HiChIP chromatin loops. Most biological; costly experimentally.

程式碼

實作範例

library(Signac); library(Seurat)
mu <- CreateSeuratObject(counts = rna_counts, assay = "RNA")
mu[["ATAC"]] <- CreateChromatinAssay(counts = atac_counts, sep = c(":", "-"),
                                          fragments = "fragments.tsv.gz")

# Peak-gene linking
mu <- LinkPeaks(mu, peak.assay = "ATAC", expression.assay = "RNA",
                   genes.use = c("GATA1", "TAL1"))
CoveragePlot(mu, region = "GATA1", features = "GATA1",
             expression.assay = "RNA", peaks = TRUE)
library(ArchR)
proj <- addPeak2GeneLinks(ArchRProj = proj,
                            reducedDims = "IterativeLSI",
                            useMatrix = "GeneExpressionMatrix")
p2g <- getPeak2GeneLinks(proj, corCutOff = 0.45, resolution = 1)
plotPeak2GeneHeatmap(ArchRProj = proj, groupBy = "Clusters")
