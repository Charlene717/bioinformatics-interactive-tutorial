為什麼重要

為什麼「選對平台」是 80% 的成功？

空間轉錄組學是一個「實驗設計決定上限」的領域——後續再厲害的演算法也無法補回平台本身的物理限制。常見的「悲劇」：

  • 研究 單細胞 T-tumor 直接接觸，卻用了 55 μm 的 Visium → 每個 spot 都混了 5-10 種細胞，怎麼算「接觸」？
  • 研究 未知的代謝重塑，卻買了 1000-gene 的 MERFISH panel → panel 裡沒有那條代謝路徑的關鍵基因。
  • 研究 15 年前歸檔的 FFPE 病理切片，卻選了只支援新鮮冷凍的 Slide-seq → 樣本全部報廢。

本章節將帶你穿越每個平台的物理 / 化學原理、規格表、價格區間，並以決策樹的形式給你 mental model。

Spatial transcriptomics is a field where experimental design caps the upside — no clever algorithm can rescue a platform's physical limits. Common tragedies:

  • Studying single-cell T-tumor contact but using 55 μm Visium → every spot mixes 5-10 cells, how do you call "contact"?
  • Studying unknown metabolic rewiring but buying a 1000-gene MERFISH panel → key metabolic genes simply aren't on the panel.
  • Studying 15-year-old archival FFPE but choosing Slide-seq (fresh-frozen only) → all samples burned.

This chapter walks you through each platform's physics / chemistry, spec sheet, and price band, then gives you a decision-tree mental model.

化學原理

兩大派別的化學原理

① Array-based：捕捉與條碼

核心步驟：

  1. 切片貼合：10 μm 厚的組織切片貼在預先排好 barcoded poly-T 寡核苷酸的玻片上。Visium 的 capture area 是 6.5×6.5 mm，含 ~5,000 個 55 μm spot。
  2. 透化（permeabilization）：用 pepsin / 蛋白酶讓細胞膜變透，mRNA 從細胞釋放後落到下方 spot。
  3. 反轉錄：poly-T 與 mRNA 的 polyA 配對，反轉錄酶如 Maxima H Minus RTase — 在玻片表面做「在位」反轉錄，把每個 spot 的 barcode 加到 cDNA 上。 把 mRNA 變 cDNA，spot 的 barcode 被「燒」進每條 cDNA。
  4. 建庫定序：cDNA 從玻片上洗下、做 NGS library、上 Illumina 機器。每條 read 含三段：spot barcode + UMI + cDNA 序列。
  5. 對位（registration）：軟體把每個 spot 的座標對齊到事先拍好的 H&E 影像上——所以你需要 H&E 影像！

Core steps:

  1. Tissue placement: 10 μm-thick section placed on a slide pre-arranged with barcoded poly-T oligos. Visium capture area is 6.5×6.5 mm with ~5,000 55 μm spots.
  2. Permeabilization: pepsin / protease permeabilizes membranes; released mRNA falls onto the spot beneath.
  3. Reverse transcription: poly-T base-pairs with mRNA polyA; reverse transcriptasee.g., Maxima H Minus RTase — performs in-place RT on the slide, attaching each spot's barcode onto the cDNA. writes cDNA with the barcode burned in.
  4. Library + sequencing: cDNA stripped from slide, NGS-libraried, run on Illumina. Each read = spot barcode + UMI + cDNA sequence.
  5. Registration: software aligns each spot to a pre-acquired H&E image — so you must take an H&E first!
關鍵：透化時間

不同組織需要不同透化時間（10x 的 Tissue Optimization kit 就是用來測這個）。腦 ~12 分鐘、肝 ~30 分鐘、肌肉 ~40+ 分鐘。透化不足 → UMI 數低；透化過頭 → mRNA 擴散到鄰近 spot，「diffusion」造成解析度下降。

Different tissues need different permeabilization times (10x's Tissue Optimization kit measures exactly this). Brain ~12 min, liver ~30 min, muscle ~40+ min. Under-permeabilized → low UMI; over-permeabilized → mRNA diffuses to neighbors and resolution degrades.

② Imaging-based：組合編碼

核心邏輯：把「基因身分」用「多輪 ON/OFF 編碼」記下來。以 MERFISH 為例（其他 imaging 平台是變種）：

  1. 探針設計：每個基因設計 30-50 條 ~30 nt 的「encoding probes」貼到 mRNA 上。每條 encoding probe 帶有「readout sequence」，這就是基因的編碼。
  2. 輪流偵測：例如 16 輪 hybridization，每輪用一條螢光 readout probe 來「讀readout probe 是 ~20 nt 短螢光探針，互補於 encoding probe 上的特定 readout sequence。每輪換不同 readout，就能在不同輪「亮」起不同的基因。」一組 readout sequence。每個基因被設計為在 16 輪中固定 4 輪「亮」、12 輪「暗」——這是它的 4-of-16 Hamming code。
  3. 影像處理：把所有輪的影像配準、做點偵測（spot calling）、把每個點的 ON/OFF 序列解碼成基因身分。錯誤校正：Hamming distance ≥ 4 能容忍 1 bit 翻轉。
  4. cell segmentation：用 DAPI（核）或細胞膜染色把點分配到細胞——這一步常是最大誤差源。

Core logic: record gene identity as a multi-round ON/OFF code. Using MERFISH (others are variants):

  1. Probe design: 30-50 ~30-nt "encoding probes" per gene hybridize to mRNA. Each carries a "readout sequence" — the gene's code.
  2. Rounds: e.g., 16 rounds of hybridization. Each round uses one fluorescent readout probeA ~20-nt short fluorescent probe complementary to a specific readout sequence on encoding probes. Different readouts each round → different genes "light up" each round.. Each gene's encoded to be "on" in a fixed 4 of 16 rounds — its 4-of-16 Hamming code.
  3. Image processing: register all rounds, do spot calling, decode each spot's ON/OFF sequence to gene identity. Error correction: Hamming distance ≥ 4 tolerates 1-bit flips.
  4. Cell segmentation: assign spots to cells via DAPI (nucleus) or membrane stain — often the largest source of error.
比喻：摩斯密碼點亮的螢火蟲

想像漆黑山谷裡有上萬隻螢火蟲，每隻代表一個 mRNA 分子。你拿著相機連拍 16 張，每張之間切換不同濾鏡（readout probe）。基因 A 的螢火蟲只在第 1, 3, 7, 12 張亮；基因 B 只在 2, 5, 9, 14 張亮——每隻螢火蟲的「閃爍序列」就是它的基因身份。MERFISH 就是用這個邏輯，把 ~140 個 mRNA 種類用 16 輪壓進一張影像。

Imagine a dark valley with thousands of fireflies — each is an mRNA molecule. You shoot 16 photos, switching filters (readout probes) between each. Gene A's fireflies are visible only in shots 1, 3, 7, 12; gene B only in 2, 5, 9, 14 — each firefly's blink pattern is its gene identity. MERFISH compresses ~140 mRNA species into 16 rounds with this logic.

真實案例

案例：DCIS 乳癌侵犯前緣

Janesick et al. 2023 Cell — 多平台乳癌地圖
2023 · Cell · 三平台比較

實驗設計：對同一塊 FFPE 乳癌組織，同時跑 Xenium（313-gene panel，亞細胞）、Visium（55 μm，全轉錄組）、CytAssist（FFPE-friendly Visium）。研究目的：(1) 看 DCIS（原位癌）到 IDC（侵犯性導管癌）的轉變地圖，(2) 比較三平台優劣。

結果 1：DCIS-IDC 邊界的「過渡帶」 在 Xenium 中清楚看到，每個過渡細胞同時表達 DCIS marker（KRT5, KRT15）與 IDC marker（KRT8, KRT18），這在 Visium 的 55 μm spot 裡會被「平均掉」。

結果 2：Visium 找的腫瘤分區與 Xenium 一致，但 Xenium 的 313 基因 panel 漏掉 了 Visium 找到的好幾個 SVG（如 SERPINA3）——可見全轉錄組的不可替代性。

結論：兩者互補。當預算允許時，最強策略是 Visium 做探索、Xenium 做亞細胞精修。

Design: on the same FFPE breast tumor, ran Xenium (313-gene panel, sub-cellular), Visium (55 μm, whole-Tx), and CytAssist (FFPE-friendly Visium). Goals: (1) map the DCIS-to-IDC transition, (2) compare platforms.

Result 1: DCIS-IDC transition zone was clearly visible in Xenium — each transitional cell co-expressed DCIS markers (KRT5, KRT15) and IDC markers (KRT8, KRT18). This signal averages out in 55 μm Visium spots.

Result 2: Visium-identified tumor regions agreed with Xenium, but Xenium's 313-gene panel missed several Visium SVGs (e.g., SERPINA3) — confirming whole-Tx is irreplaceable.

Conclusion: the two are complementary. Budget permitting, the strongest strategy is Visium for discovery + Xenium for sub-cellular refinement.

新手陷阱

三個必避免的錯誤

陷阱 1：被「解析度」字面綁架

看到 Visium HD 標示「2 μm」就以為比 Visium 「強 27 倍」。實際上：2 μm bin 的 UMI 數常太稀，分析時要 bin 到 8 或 16 μm；而且 mRNA 擴散決定了真實有效解析度 ~3-4 μm。買 HD 不一定是最划算選擇——對中等解析度需求（如組織層），標準 Visium 已綽綽有餘。

Seeing Visium HD's "2 μm" label, people assume it's "27× better" than Visium. Reality: 2 μm bins are usually too sparse — analysis bins to 8 or 16 μm; plus mRNA diffusion caps real effective resolution at ~3-4 μm. HD isn't always the smart buy — for moderate-resolution needs (tissue layers), standard Visium is plenty.

陷阱 2：選 panel 卻沒檢查內容

買 MERFISH/Xenium 之前務必逐基因檢查 panel。MERFISH 預設 panel 多偏向神經科學；Xenium 的 5K human pan-tissue panel 偏向常見癌症免疫 marker。若你的研究是「肝臟代謝重塑」，可能要付額外費用做 custom panel（多花 $10k+）。

Before buying MERFISH/Xenium, check the panel gene-by-gene. MERFISH presets lean neuroscience; Xenium 5K human pan-tissue lean common cancer-immune markers. For "liver metabolic rewiring", you may need a custom panel (+$10k easily).

陷阱 3：FFPE 質量沒先檢查

FFPE 切片的 RNA 完整性（DV200DV200 = RNA 中片段長度 ≥ 200 nt 的比例。Visium FFPE 推薦 DV200 ≥ 30%；< 20% 通常失敗。）差異極大。10 年前的切片 DV200 可能 < 20%，跑 Visium FFPE 會失敗。送樣前先抽 RNA 跑 Tapestation，確認 DV200 ≥ 30%。

FFPE RNA integrity (DV200DV200 = fraction of RNA fragments ≥ 200 nt. Visium FFPE recommends DV200 ≥ 30%; < 20% usually fails.) varies enormously. A 10-year-old block may have DV200 < 20% and Visium FFPE will fail. Extract RNA and run Tapestation first to confirm DV200 ≥ 30%.

分層理解

按程度認識「探針化學」

Visium 用 poly-T 探針——只能抓有 polyA 尾巴的 mRNA（占 ~90% 的 mRNA）。FFPE 樣本因為 RNA 斷裂、polyA 常掉，Visium 標準款做不了。

2022 年推出的 Visium FFPE 改用 探針對每基因 2 條 ~50 nt 的 LHS+RHS 探針，貼在同一條 mRNA 上後做 ligation 才能變 cDNA——這選擇性只連到「兩端都還在」的 mRNA 片段。 連接策略，可在斷裂 RNA 上工作。

Visium uses poly-T probes — capturing only mRNA with polyA tails (~90% of mRNA). FFPE samples have fragmented, polyA-stripped RNA, so standard Visium fails.

The 2022-released Visium FFPE switches to probe pairsTwo ~50-nt LHS+RHS probes per gene; both must bind the same mRNA and be ligated into cDNA. This selects only mRNA fragments still intact at both probe sites. with ligation chemistry, working on degraded RNA.

探針對策略（Visium FFPE、Xenium、CosMx 都用）有兩個關鍵：

  • 特異性高：需 LHS + RHS 兩條同時對到才連，誤標機會大降。
  • panel 受限：每基因要設計 specific probe pair，無法做 unbiased 全轉錄組。

Xenium 用「padlock probe」加 rolling circle amplification (RCA)——讓單一 mRNA 變成幾百倍亮的訊號，比 MERFISH 的 encoding probe 更亮但要多兩步酶反應。

Probe-pair strategy (used by Visium FFPE, Xenium, CosMx) has two implications:

  • High specificity: both LHS and RHS must bind for ligation — misassignment rate drops sharply.
  • Panel limited: each gene needs a designed probe pair — no unbiased whole-Tx.

Xenium uses padlock probes + rolling circle amplification (RCA) — turning one mRNA into a hundred-fold brighter signal vs MERFISH's encoding probes; two extra enzymatic steps trade off.

編碼空間（codebook）的設計考量：

  • Hamming distance ≥ 4：容忍 1-bit 翻轉。MERFISH 用 4-of-16 binary code，可編碼 C(16,4) = 1820 個基因，但實際只用 ~140-1000，留下「空 code」做 negative control 偵測 cross-hybridization。
  • Code 分布要均衡：每輪 ON 的基因數要平均（避免某輪曝光率特別高），用最佳化演算法（如 simulated annealing）分配。
  • Decoding 後處理：常見策略是 max-likelihood 解碼 + spot 強度 z-score 過濾 + 跨輪 registration error 量化。

Xenium 與 MERFISH 的關鍵差異：Xenium 訊號被 RCA 放大 ~100×，所以單張影像即可解碼一輪；MERFISH 每點亮度低、需要 confocal 高 NA 物鏡才能看清。這也是為什麼 Xenium 更適合常規實驗室、MERFISH 偏實驗室自製（DIY）。

Codebook design considerations:

  • Hamming distance ≥ 4: tolerates 1-bit flips. MERFISH's 4-of-16 binary code can encode C(16,4) = 1820 genes, but practically uses 140-1000 — empty codes serve as negative controls for cross-hybridization detection.
  • Balanced code distribution: "ON" count per round should be balanced (avoid one round hot) — solved by simulated annealing or ILP.
  • Decoding post-processing: ML decoding + spot-intensity z-score filtering + registration-error quantification across rounds.

Key Xenium vs MERFISH difference: Xenium signal is ~100× RCA-amplified, so one image per round decodes; MERFISH spots are dim, requiring high-NA confocal. This is why Xenium suits routine labs while MERFISH skews DIY academic.

Very useful, but panels cap at 60-100 antibodies and antibody choice is limited. Best use: same-section RNA + key proteins (e.g., PD-L1, HLA surface levels) — especially in immuno-oncology where protein is functionally closer than mRNA.

Viable but the bar is high. Needs: (1) high-NA confocal, (2) automated fluidics (perfusion), (3) TB-scale storage, (4) image-processing expertise. From scratch: ~$500k+ in equipment + staff. Cheaper to buy Vizgen MERSCOPE (commercial MERFISH) or Xenium if budget-limited.

開放性問題
