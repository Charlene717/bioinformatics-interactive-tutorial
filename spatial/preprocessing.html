為什麼重要

為什麼空間 QC 不同？

scRNA-seq 的 QC 假設：「異常的 cell barcode = 壞細胞 = 該丟掉」。空間轉錄組學的 QC 必須拋棄這個假設，因為：

  1. 「異常 spot」可能是真實生物訊號：腫瘤的壞死核心本就 mt% 高、UMI 低，但這是真實的腫瘤分區，不該丟。丟了你就看不到「壞死區包圍腫瘤核心」這個重要結構。
  2. spot 不是細胞：一個 Visium spot 是 ~5,000 個 spot 中的一個，本身就是「混合多細胞訊號」。傳統「單細胞 QC 思維」不適用。
  3. 位置本身就是 QC 變量：邊緣 spot 通常 UMI 低（組織少）、中央 spot UMI 高——你不能用同一個 threshold 對待邊緣與中央。
  4. 多切片批次效應：不同實驗日、不同切片厚度、不同 RNA quality 都會造成 spot 間 UMI 差距大；需要 batch-aware normalization。

scRNA-seq QC assumes: "abnormal barcode = bad cell = discard". Spatial transcriptomics must abandon this because:

  1. "Abnormal spots" can be real biology: a tumor's necrotic core naturally has high mt% and low UMI — but it's a true tumor zone. Drop it and you lose the "necrosis encircling tumor core" architecture.
  2. A spot is not a cell: each Visium spot is one of ~5,000 capture areas — inherently a mixed signal. Traditional "single-cell QC mindset" doesn't transfer.
  3. Position itself is a QC covariate: edge spots have low UMI (less tissue); central spots are high — you can't apply one threshold equally.
  4. Multi-section batch effects: different experimental days, section thicknesses, RNA qualities all create spot-level UMI gaps — needs batch-aware normalization.
比喻：城市衛星照

想像看一張夜間衛星照判斷城市繁榮度——亮的地方一定繁榮嗎？不一定。夜間機場、墓園、公園都暗，但都很重要。如果你只憑「光強度 > threshold」過濾，會把這些「結構性暗區」當雜訊丟掉。空間 QC 也是——壞死區是「結構性暗區」，要保留、要標註，而不是粗暴過濾。

Imagine judging a city's prosperity from a nighttime satellite image. Bright = prosperous? Not always. Airports, cemeteries, parks are all dark but important. If you filter by "brightness > threshold", you trash these "structural dark zones" as noise. Spatial QC is similar — necrotic zones are "structural darkness", to be preserved and annotated, not bluntly filtered.

SpaceRanger 輸出剖析

10x Visium 的官方流程 SpaceRanger 會把每個樣本輸出為一個資料夾。理解每個檔案的角色，才能在自己的 pipeline 中靈活運用：

10x's official SpaceRanger pipeline outputs one folder per sample. Understanding each file's role is essential for flexible pipelines:

sample_outs/
├─ filtered_feature_bc_matrix.h5    # 主表達矩陣（HDF5）— 只含「在組織內」的 spots
├─ raw_feature_bc_matrix.h5          # 未過濾矩陣（所有 ~5,000 spots，含組織外）
├─ filtered_feature_bc_matrix/       # 同上但 mtx 格式（matrix.mtx, features.tsv, barcodes.tsv）
├─ spatial/
│  ├─ tissue_positions.csv          # spot ID, in_tissue 旗標, array_row, array_col, 像素 x/y
│  ├─ tissue_lowres_image.png       # 低解析縮圖（一般 600 px）— 顯示用
│  ├─ tissue_hires_image.png        # 高解析 H&E（~2000 px）— 出版用
│  ├─ detected_tissue_image.jpg     # 演算法偵測到「組織區」的遮罩
│  ├─ scalefactors_json.json        # 像素 ↔ μm 換算；不同影像版本有不同 scale
│  └─ aligned_fiducials.jpg          # 對位用的 fiducial markers（玻片角的小圖案）
├─ analysis/                         # SpaceRanger 自帶 PCA、clustering、DE
├─ web_summary.html                  # 樣本層級 QC 報告（必看！）
├─ metrics_summary.csv               # web_summary 同樣指標的 CSV 版
├─ molecule_info.h5                  # 分子層級資料（重新分析或合併樣本時用）
└─ possorted_genome_bam.bam          # 比對到基因組的 reads（除錯用）
關鍵：scalefactors_json.json

這個小檔有三個 scale：tissue_hires_scalef（hires 影像對原始圖的比例）、tissue_lowres_scalef（lowres 對原始圖比例）、spot_diameter_fullres（spot 在 fullres 影像下的直徑像素）。當你寫自己的 spot ↔ μm 轉換時，永遠要用對應 scale，不然位置會錯位 10×！

This tiny file holds three scales: tissue_hires_scalef (hires image vs original), tissue_lowres_scalef (lowres vs original), spot_diameter_fullres (spot diameter in full-res pixels). When writing custom spot↔μm code, always use the matching scale — wrong scale causes 10× position misalignment!

陷阱：filtered vs raw 弄錯

filtered_feature_bc_matrix.h5 只含 SpaceRanger 判定「組織內」的 spot；raw_feature_bc_matrix.h5 含全部 spot（含組織外的）。日常分析用 filtered；做 ambient RNA 校正、空 spot 訊號分析時才用 raw。新手常一律用 raw → 結果有大量 0-UMI 空 spot 把分析弄歪。

filtered_feature_bc_matrix.h5 contains only spots SpaceRanger called "in-tissue"; raw_feature_bc_matrix.h5 contains all spots (including empty/out-of-tissue). Routine analysis uses filtered; raw is for ambient-RNA correction or empty-spot analysis. Beginners often use raw blindly → analysis dragged by zero-UMI empty spots.

標準化

空間標準化

空間樣本標準化大致同 scRNA-seq，但需注意：

Largely the same as scRNA-seq, with caveats:

SCTransform v2

原理：對每個基因做 NB 回歸（regress out 測序深度），把殘差當「標準化值」。

適用：Visium 預設、組織異質性強。
注意：v2 對低基因數樣本更穩定。

How: per-gene NB regression (regress out depth); use residuals as normalized values.

For: Seurat Visium default; strong tissue heterogeneity.
Note: v2 more stable on low-feature samples.

LogNormalize

原理：對每 spot 除總 UMI、乘 10,000、log1p。簡單透明。

適用：跨工具一致；下游有 Harmony 等 batch correction 時穩定。

How: divide by total UMI per spot × 10,000, then log1p. Simple and transparent.

For: cross-tool consistency; stable with downstream batch correction (Harmony).

Pearson residuals (analytic Pearson)

原理：Lause & Berens 2021 提出，比 SCTransform 快 100×，數學上更乾淨。

適用：大型 Visium HD / Stereo-seq；scanpy 已實作。

How: Lause & Berens 2021; ~100× faster than SCTransform with cleaner math.

For: large Visium HD / Stereo-seq; implemented in scanpy.

SME (Spatial Morphological gene Expression)

原理：stLearn 提出，用 H&E 影像萃取的 morphology 特徵 + 鄰居 spot 平均做 smoothing。

適用：低 UMI 樣本想做空間平滑時。

How: stLearn — smooth via H&E-derived morphology features + neighbor-spot averaging.

For: low-UMI samples that benefit from spatial smoothing.

⚠️
千萬不要做的事：用 deconvolution 出來的 cell-type 比例直接餵 DESeq2 / edgeR——proportions 是 softmax 機率，不是 raw counts，模型假設完全不成立。要做差異就在 cluster / 區域層級做 pseudobulk。
Never do this: feed deconvolution-derived cell-type proportions to DESeq2/edgeR — proportions are softmax probabilities, not counts; model assumptions are violated. For DE, use pseudobulk at cluster/region level.
真實案例

案例：膠質母細胞瘤壞死核心

Ravi et al. 2022 Cancer Cell — GBM
28 GBM Visium 切片

故事：研究團隊一開始套用標準 QC（mt% > 25% 全部丟掉），結果發現「壞死核心整片消失」——但他們的研究問題正是「壞死如何驅動腫瘤異質性」！

解法：改寫流程：(1) 不丟高 mt% spot，而是把它們標註為 region = necrotic；(2) 用空間鄰接資訊識別「壞死核心 ↔ 缺氧 ring ↔ 增殖前緣」三層結構；(3) 在每一層內做 DE。

關鍵發現：缺氧 ring 富含 VEGFA, HIF1A, CA9——血管新生訊號；增殖前緣富含 MKI67, TOP2A。這個分區只有「不過嚴的 QC + 空間註釋」才能看見。

教訓：把 QC 當「標註步驟」而非「丟棄步驟」。先標 region，再決定哪些 region 進入哪些下游分析。

Story: the team initially applied standard QC (drop mt% > 25%) and found the necrotic core entirely disappeared — but their question was literally "how does necrosis drive heterogeneity"!

Fix: rewrote pipeline: (1) don't drop high mt% spots — label them region = necrotic; (2) use spatial adjacency to identify a 3-layer structure: necrotic core ↔ hypoxic ring ↔ proliferative front; (3) DE within each layer.

Key finding: hypoxic ring is rich in VEGFA, HIF1A, CA9 — angiogenesis; proliferative front in MKI67, TOP2A. This zonation is only visible with lenient QC + spatial annotation.

Lesson: treat QC as annotation, not deletion. Tag regions first; decide later which regions enter which downstream analyses.

多切片整合

多切片整合

實際研究常有「3 個治療組 × 2 個切片 / 組 = 6 切片」的場景。整合多切片時：

  • 對齊（registration）：用 PASTE / STalign 把多切片放到「共享座標系統」，跑跨切片的 SVG / region 比較。
  • 批次校正：表達矩陣層面用 Harmony / scVI，把不同實驗日的影響消除。
  • QC 一致性：每個 sample 用自己的 MAD 自適應門檻，而非全研究用同一閾值。

Real studies often have "3 treatments × 2 sections each = 6 sections". When integrating:

  • Registration: use PASTE / STalign to put sections in a shared coordinate system for cross-section SVG / region comparison.
  • Batch correction: on the expression matrix, use Harmony / scVI to remove experimental-day effects.
  • Consistent QC: use per-sample MAD-adaptive thresholds, not a single study-wide threshold.

新手陷阱

四個必避免的錯誤

陷阱 1：誤殺壞死區

壞死區 mt% 高、UMI 低——但這是真實生物。用 mt% > 25% 一刀切會丟掉壞死、缺氧、發炎核心。建議：標註而非丟棄。

Necrotic zones have high mt%, low UMI — but it's real biology. Slashing mt% > 25% deletes necrosis, hypoxia, inflammatory cores. Better: annotate, don't delete.

陷阱 2：統一閾值

不同實驗日、不同切片厚度、不同 RNA quality → UMI 分布天差地遠。用全研究一個閾值會把某些「正常」sample 全切掉。建議：用 scater::isOutlier() 或 MAD-based 自適應門檻。

Different days, section thickness, RNA quality → UMI distributions diverge. One global threshold can wipe certain "normal" samples. Better: use scater::isOutlier() or MAD-based adaptive thresholds.

陷阱 3：忘了血污

大血管附近、出血區的 spot 會富含 HBA、HBB——這些不是壞品質，但會「淹沒」其他訊號。建議：計算 percent.hb，標註血管區，但不一定全丟。

Spots near large vessels or hemorrhages are rich in HBA, HBB — not low quality but they "drown" other signals. Better: compute percent.hb, label vessel zones, but don't always delete.

陷阱 4：raw vs filtered 弄錯

日常 EDA 用 filtered（只組織內）；做 ambient RNA 校正、空 spot 訊號分析才用 raw。新手常一律用 raw → 大量 0-UMI spot 拖累分析。

Routine EDA uses filtered (in-tissue only); ambient-RNA correction and empty-spot analysis use raw. Beginners default to raw → analysis bogged down by zero-UMI spots.

常見問題

QC FAQ

完全看組織。神經元 ~5%、心肌 ~15-30%、肝 ~10-15%、腫瘤壞死區 ~30-50%。用「全組織 MAD」做自適應比固定 25% 更合理。心肌切片強行套 5% 會把所有 spot 殺光。

Totally tissue-dependent. Neurons ~5%, cardiomyocytes ~15-30%, liver ~10-15%, tumor necrosis ~30-50%. MAD-adaptive over the whole section is more rational than a fixed 25%. Forcing 5% on cardiac sections kills every spot.

Visium 預設 SCTransform v2，組織異質性強時表現好；若要跨工具一致或下游有 Harmony，LogNormalize 更穩。大型 Visium HD / Stereo-seq → 用 Pearson residuals（scanpy），快 100×。

Visium default is SCTransform v2 — strong with heterogeneous tissue; if cross-tool consistency or downstream Harmony, LogNormalize is more stable. Large Visium HD / Stereo-seq → Pearson residuals (scanpy), ~100× faster.

不建議全移除——它們含真實生物訊號（蛋白合成活性）。但在挑 HVG/SVG 時可以排除 RPS/RPL，避免「核糖體基因主導」變異。Seurat 與 scanpy 都有 regress / exclude pattern 選項。

Don't fully remove — they carry real biology (protein-synthesis activity). But when picking HVG/SVG, excluding RPS/RPL avoids "ribosomal-dominated variance". Both Seurat and scanpy provide regress/exclude options.

官方建議在 8 μm 或 16 μm bin 層做 QC（2 μm 太稀無法穩定計算 mt%）。10x 的 spaceranger 已自動產生多 bin 層級的 h5 檔，選 8 μm 開始最實用。

Official advice: QC at the 8 μm or 16 μm bin level (2 μm is too sparse for stable mt%). 10x's spaceranger auto-generates multi-bin h5 files — start with 8 μm.

Xenium 用 xeniumranger；Vizgen MERSCOPE 用內建 MERlin pipeline。輸出皆為 transcript-level (x, y, gene) CSV + cell segmentation mask。QC 重點是 segmentation 品質、 false-positive rate（每基因 blank code 偵測）、cell shape。

Xenium uses xeniumranger; Vizgen MERSCOPE uses its built-in MERlin pipeline. Both output transcript-level (x, y, gene) CSV + cell segmentation masks. Key QC: segmentation quality, false-positive rate (per-gene blank-code detection), cell shape.

表達矩陣層用 Harmony（最常用）或 scVI（GPU 加速）；空間座標層用 PASTE 或 STalign。注意：批次校正不能矯枉過正——若不同治療組真的有不同表達模式，過度 batch correction 會把治療效應也消除。

For expression: Harmony (most common) or scVI (GPU-accelerated); for coordinates: PASTE or STalign. Caveat: don't over-correct — if treatment groups have genuinely different expression, aggressive batch correction also erases the treatment effect.

深度思考

開放問題
  1. 你拿到一個 GBM Visium 樣本，UMI 中位數 800、mt% 中位數 22%。標準 QC 會建議「丟掉太多 spot」，但生物上你預期看到壞死。你會怎麼設計過濾策略？

  3. 兩個 Visium 樣本，sample A 的 spot UMI 中位數是 sample B 的 3 倍。你會直接合併分析嗎？需要做什麼處理？
  4. You have a GBM Visium sample: median UMI 800, median mt% 22%. Standard QC would drop too many spots, but biologically you expect necrosis. How do you design filtering?
  5. Why is "spatial autocorrelation of housekeeping genes" a useful QC metric? What's the biological logic?
  6. Two Visium samples: A's median UMI is 3× B's. Would you merge directly? What needs to happen first?
