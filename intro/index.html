生物資訊前導介紹

在打開任何 pipeline 之前——先把「生物資訊到底在做什麼」想清楚。從 DNA → 定序 → 檔案 → 分析 → 生物學洞見。

Before opening any pipeline — let's get clear on what bioinformatics actually is. From DNA → sequencing → files → analysis → biological insight.

開場故事

為什麼要學生物資訊？

1990 年代人類基因組計畫花了 13 年、30 億美元 才定序出 1 個人類基因組；今天，一台 Illumina NovaSeq X 一天 就能定序 20 個全基因組，成本不到 USD 200。資料量爆炸但問題也跟著放大——怎麼從一座一座的 TB 級 FASTQ 山裡找出有意義的生物學？這就是生物資訊（bioinformatics）的工作。

本教學從體學分析最常見的工作流出發：raw reads → QC → 比對 → 量化 → 統計 → 生物學詮釋。掌握這條主軸後，所有體學（基因體、轉錄體、表觀基因體、蛋白體、微生物體）只是「換資料、換工具、共享邏輯」。

The Human Genome Project took 13 years and ~US $3 billion in the 1990s to sequence one genome. Today, a single Illumina NovaSeq X can produce ~20 whole human genomes per day for under US $200 each. Data explosion brought new problems: how do we extract meaningful biology from terabytes of FASTQ? That's bioinformatics.

This tutorial follows the canonical omics workflow: raw reads → QC → alignment → quantification → statistics → biological interpretation. Once this backbone is clear, every omics (genome, transcriptome, epigenome, proteome, microbiome) is "same logic, different data, different tools."

第一性原理

中心法則 (Central Dogma) — 一切的起點

DNA→ 轉錄 (transcription) RNA→ 轉譯 (translation) Protein

每一種體學量測，本質上都是切下「中心法則」中某一個層級或介面：

  • 基因體學 (Genomics)：讀 DNA 序列——個體間有何變異？哪些變異致病？
  • 表觀基因體學 (Epigenomics)：讀染色質狀態與 DNA 修飾——哪些基因「可以被表達」？
  • 轉錄體學 (Transcriptomics)：讀 RNA 表達量——哪些基因「實際被表達」、多少？
  • 蛋白體學 (Proteomics)：讀蛋白質豐度與修飾——細胞「實際在做什麼」？
  • 代謝體學 (Metabolomics)：讀代謝物——細胞功能的最終讀數。

每往中心法則下游走一層，資訊量增加（複雜度↑）但訊號穩定度下降——這是體學分析的核心 trade-off。

Each omics measurement essentially slices a layer (or an interface) of the central dogma:

  • Genomics: DNA sequence — what varies between individuals? Which variants cause disease?
  • Epigenomics: chromatin state & DNA modifications — which genes are eligible for expression?
  • Transcriptomics: RNA levels — which genes are actually expressed, and how much?
  • Proteomics: protein abundance and modifications — what the cell is doing.
  • Metabolomics: small-molecule metabolites — the functional readout of cellular activity.

Each downstream layer adds biological information but loses signal stability — this is the central trade-off of multi-omics.

技術核心

定序原理：四大平台對照

平台 化學原理 讀長 / 準確度 典型用途
SangerddNTP 連鎖終止~800 bp / Q40+變異確認、小片段定序
Illumina (NovaSeq / NextSeq)橋式擴增 + SBS2×150 bp / Q30+WGS、WES、RNA-seq、ATAC（短讀主流）
PacBio HiFi (Revio)SMRT + CCS 共識15-25 kb / Q30+ (HiFi)長讀 WGS、結構變異、isoform、HLA 分型
Oxford Nanopore (R10.4.1, Dorado duplex)奈米孔通過電流變化10 kb-1 Mb+ / Q20-Q30 (duplex)超長讀、即時定序、直接讀取甲基化
💡
第一個關鍵分辨：短讀（Illumina）每個 read 約 150 bp，準確但無法跨越重複區；長讀（PacBio、Nanopore）每個 read 數萬至百萬 bp，可解析結構變異與長 RNA isoform，但歷史上錯誤率較高（HiFi 與 duplex 已修正）。
The first dichotomy: short reads (Illumina) — ~150 bp, highly accurate, struggles with repeats. Long reads (PacBio, Nanopore) — kb to Mb, resolves structural variants and full isoforms, historically lower per-base accuracy (now solved by HiFi and duplex).
關鍵概念

Reference Genome——你比對的「底圖」

定序得到的 raw reads 像是把一本書撕成 10 億片碎紙；要還原原文，你需要一本「答案參考書」——這就是 reference genome。

  • GRCh38 / hg38：2013 年迄今的人類標準參考基因組，含 ALT contigs（額外的單倍型）與 decoy 序列（避免錯誤比對）。
  • T2T-CHM13 (2022)：第一個「telomere-to-telomere」完整人類基因組——包含著絲粒、rDNA、Y 染色體完整序列，比 GRCh38 多 5% 序列。
  • Pangenome (HPRC, 2023)：不再用單一參考；用 47+ 個樣本建立 graph genome，減少 reference bias。

選對 reference 很重要：對東亞族群做 WGS，用 T2T-CHM13 或加入族群特異變異能避免 reference bias 造成的假陰性。

Raw reads are like a book torn into a billion confetti pieces; to reconstruct the original, you need an answer key — the reference genome.

  • GRCh38 / hg38: The standard human reference since 2013; includes ALT contigs (alternate haplotypes) and decoy sequences to avoid misalignment.
  • T2T-CHM13 (2022): The first telomere-to-telomere complete human genome — covers centromeres, rDNA, and the full Y chromosome; adds 5% sequence over GRCh38.
  • Pangenome (HPRC, 2023): No single reference; a graph genome built from 47+ samples to reduce reference bias.

Choosing the right reference matters: for East-Asian WGS, using T2T-CHM13 or adding population-specific variants prevents false-negative calls from reference bias.

檔案地圖

體學六大檔案格式

🧬

FASTQ

@read_id + 序列 + + + Phred 品質字串。每個 read 4 行；定序機原始輸出格式。

@read_id + sequence + + + Phred quality. 4 lines per read; raw sequencer output.

🗺️

SAM / BAM / CRAM

read 對齊到 reference 後的結果。BAM 是 SAM 的壓縮二進位版；CRAM 進一步用 reference 壓縮，省 50% 空間。

Alignment of reads to a reference. BAM is the compressed binary form of SAM; CRAM compresses further by referencing the genome, saving ~50% space.

⚠️

VCF

Variant Call Format。儲存單核苷酸變異 (SNV)、insertion/deletion、結構變異的標準格式。

Variant Call Format — stores SNVs, indels, and structural variants in a standardised tab-delimited form.

📏

BED / BigWig

BED：基因組區間（chr + start + end）。BigWig：每個位點的數值訊號（如 coverage、ChIP signal）。

BED: genomic intervals (chr + start + end). BigWig: per-position numeric signal (coverage, ChIP signal).

📝

GTF / GFF3

基因註釋——告訴你哪些 chr + 位置是基因、轉錄本、外顯子、UTR、CDS。

Gene annotations — tells you which chromosomal regions are genes, transcripts, exons, UTRs, CDSs.

📦

HDF5 / AnnData / MTX

表達矩陣（細胞×基因或樣本×基因）。AnnData 是 scRNA 的主流格式；HDF5/MTX 是 10x 與 Bioconductor 常見輸出。

Expression matrices (cell × gene or sample × gene). AnnData is the de-facto scRNA format; HDF5/MTX are common in 10x Cell Ranger and Bioconductor.

通用流程

八步驟通用體學流程

① QC 品質檢查

FastQC + MultiQC 看 Phred 分數、adapter、過度代表序列；garbage in, garbage out。

② Trimming 修剪

fastp / Trimmomatic / cutadapt 去除 adapter 與低品質鹼基。

③ Alignment 對齊

BWA-MEM2（DNA）/ STAR（剪接 RNA）/ minimap2（長讀）/ Bowtie2（ATAC、ChIP）。

④ Post-alignment QC

markdup（PCR 重複移除）、BQSR（鹼基品質校正，DNA）、coverage 統計。

⑤ 量化／呼叫

Salmon / featureCounts（RNA）、HaplotypeCaller / DeepVariant（DNA）、MACS2/3（ATAC/ChIP）。

⑥ 統計建模

DESeq2 / edgeR / limma-voom（差異表達）、VQSR / Mutect2（變異過濾）。

⑦ 註釋與功能

VEP / SnpEff / ANNOVAR（變異註釋）、clusterProfiler / GSEA（路徑分析）。

⑧ 視覺化與報告

ggplot2 / matplotlib / IGV / UCSC——把數字變成可說故事的圖。
