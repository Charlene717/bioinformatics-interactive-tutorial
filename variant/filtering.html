🟢 Hard Filter 🟡 VQSR 直覺 🔴 Gaussian Mixture 🔵 CNN / DRAGEN
為什麼重要

原始 VCF 的 FP 真相

未經過濾的 HaplotypeCaller 原始 VCF，SNV FP 可達 1–5%，indel FP 高達 5–15%。對族群 100 萬個變異而言，這代表 5–15 萬個假陽性。對個別臨床檢測，這直接決定 VUS 是否暴增。

本章拆解兩條主流過濾路線：

  • VQSRVariant Quality Score Recalibration — GATK 用 Gaussian mixture 學 truth set vs novel 變異的分布差異，給每個變異一個 VQSLOD 分數。 (Variant Quality Score Recalibration)：機器學習，需 ≥ 30 樣本
  • Hard Filter用固定門檻 (QD, FS, MQ, SOR ...) 篩變異。簡單、可解釋，但效能稍差。：固定門檻，小樣本與非人類首選
  • CNN 過濾：GATK CNNScoreVariants，小樣本 + 人類的替代方案

Raw HaplotypeCaller VCFs have SNV FP ~1–5% and indel FP 5–15%. For a cohort with 1 M variants, that's 50–150 k FPs. For clinical testing, it directly inflates VUS counts.

This chapter dissects two mainstream filtering routes:

  • VQSRVariant Quality Score Recalibration — GATK uses a Gaussian mixture to learn truth-vs-novel distribution differences, assigning each variant a VQSLOD score. — machine learning, needs ≥ 30 samples
  • Hard FilterFixed thresholds (QD, FS, MQ, SOR, ...). Simple, interpretable, slightly worse performance. — fixed thresholds; small cohorts and non-human first choice
  • CNN filtering — GATK CNNScoreVariants, alternative for small human cohorts

VQSR is like a smart recycling machine: show it 30 samples of "known-good variants (HapMap)" and noise, it learns "good variants look like this". Hard filter is paper recycling rules: "QD < 2, throw out" — simple but can't adapt.

VQSR 拆解

VQSR Gaussian Mixture 直覺

VQSR 的核心想法：在「INFO 欄位多維空間」(QD × FS × MQ × MQRankSum × SOR × ReadPosRankSum) 中，truth set 變異 (HapMap, 1000G) 聚成一團「good cluster」；novel 變異散在各處。對每個變異算「離 good cluster 多近」(LOD score)，給每個變異一個品質。

關鍵：truth set 變異不需要全部都真，只要多數真就能定義 cluster 中心。novel 變異中，落在 cluster 內的「自動加分」、落在外圍的「自動扣分」。

VQSR core idea: in the multi-dim space of INFO fields (QD × FS × MQ × MQRankSum × SOR × ReadPosRankSum), truth-set variants (HapMap, 1000G) cluster as a "good cluster"; novel variants scatter. For each variant, compute "how close to good cluster" (LOD score) and assign a quality.

Key: truth-set variants don't need to be all real — they just need to be mostly real to define the cluster center. Novel variants inside the cluster get a boost; outside get penalized.

正式：VQSR 用 positive model 學 truth set 的 Gaussian mixture (k components, 通常 k=8)，negative model 學 ALL novel 變異的混合分布。對每變異 v：

VQSLOD(v) = log P(v | positive) − log P(v | negative)
P(v | positive) = Σk πk · N(v; μk, Σk) Higher VQSLOD → more like truth set

k 個高斯成分由 EM 演算法擬合；feature 矩陣 = (variant × INFO field)。

Formally: VQSR fits a positive model Gaussian mixture (k components, usually k=8) on truth variants; a negative model on all novel variants. For each variant v:

VQSLOD(v) = log P(v | positive) − log P(v | negative)
P(v | positive) = Σk πk · N(v; μk, Σk) Higher VQSLOD → more truth-like

k Gaussians are fit by EM; the feature matrix is (variant × INFO field).

Tranche 是「按 truth-sensitivity 切的桶」：

  • 99.9 tranche：保留 truth set 99.9% 的變異 → 寬鬆 → 高 sensitivity / 低 precision
  • 99.0 tranche：保留 truth 99% → 嚴格 → 低 sensitivity / 高 precision

GATK 建議：SNP 用 99.7 tranche（在 sensitivity 與 precision 取得平衡），Indel 用 99.0（indel 較難呼叫，要更嚴格）。Best Practice：先用 99.9 為起點，看 ROC 曲線決定最佳切點。

Tranches are "buckets cut by truth-sensitivity":

  • 99.9 tranche: keeps 99.9% of truth → lenient → high sensitivity, low precision
  • 99.0 tranche: keeps 99% of truth → strict → low sensitivity, high precision

GATK recommends: SNP at 99.7 tranche (balance), indel at 99.0 (indels are harder, demand stricter cuts). Best practice: start at 99.9 and view ROC curves to pick the optimal cut.

為什麼 ≥ 30 樣本

VQSR 為什麼要 ≥ 30 樣本？

VQSR 的成功仰賴多維高斯混合擬合穩定。樣本太少時：

  • 變異密度太低：10 樣本可能只 ~5 萬個變異點，不足以擬合 8 個高斯成分 (每個 ~6 維)
  • truth set 取樣不足：cohort 太小時，落在 truth set 的變異可能只有幾百個
  • features 變異不足：QD、FS 等的分布範圍縮小，good cluster 與 noise 重疊度高

實證：cohort < 30 跑 VQSR，model 不收斂 (EM 在 negative model 上爆掉) 或 VQSLOD 分布失常。GATK 文件建議：SNP ≥ 30 個 cohort 樣本（or single 30× WGS exome），indel ≥ 200 個。

VQSR's success depends on stable multivariate Gaussian-mixture fitting. With too few samples:

  • Low variant density: 10 samples may yield only ~50 k variants — too few to fit 8 Gaussians (~6D each).
  • Truth set undersampled: small cohorts may have only hundreds of truth-set variants.
  • Feature variance: QD, FS, etc. have narrow ranges; good cluster overlaps noise.

Empirically: cohorts < 30 fail VQSR (EM blows up on the negative model) or yield distorted VQSLOD distributions. GATK docs recommend: ≥ 30 cohort samples for SNP (or single 30× WGS exome equivalent), ≥ 200 for indels.

⚠️
替代方案：cohort < 30 → 用 hard filter（最穩定）或 CNNScoreVariants (GATK 內建 CNN，單樣本即可)。
Alternatives: cohort < 30 → use hard filter (most stable) or CNNScoreVariants (GATK's built-in CNN, single-sample OK).

程式碼

實作範例

# 拆 SNP，套 GATK 官方門檻
gatk SelectVariants -V cohort.vcf.gz \
  --select-type-to-include SNP -O cohort.snps.vcf.gz

gatk VariantFiltration -V cohort.snps.vcf.gz -O cohort.snps.flt.vcf.gz \
  --filter-expression "QD < 2.0"            --filter-name "QD2"  \
  --filter-expression "FS > 60.0"           --filter-name "FS60" \
  --filter-expression "SOR > 3.0"           --filter-name "SOR3" \
  --filter-expression "MQ < 40.0"           --filter-name "MQ40" \
  --filter-expression "MQRankSum < -12.5"   --filter-name "MQRS" \
  --filter-expression "ReadPosRankSum < -8.0" --filter-name "RPRS"

# 只留 PASS
bcftools view -f PASS cohort.snps.flt.vcf.gz -Oz -o cohort.snps.pass.vcf.gz
# Indel 門檻較寬鬆 (indel 本來就難呼叫)
gatk SelectVariants -V cohort.vcf.gz \
  --select-type-to-include INDEL -O cohort.indels.vcf.gz

gatk VariantFiltration -V cohort.indels.vcf.gz -O cohort.indels.flt.vcf.gz \
  --filter-expression "QD < 2.0"             --filter-name "QD2"  \
  --filter-expression "FS > 200.0"           --filter-name "FS200" \
  --filter-expression "SOR > 10.0"           --filter-name "SOR10" \
  --filter-expression "ReadPosRankSum < -20.0" --filter-name "RPRS20"
# 1. 訓練模型 (SNP)
gatk VariantRecalibrator \
  -V cohort.vcf.gz -R GRCh38.fa \
  --resource:hapmap,known=false,training=true,truth=true,prior=15.0 hapmap_3.3.hg38.vcf.gz \
  --resource:omni,known=false,training=true,truth=true,prior=12.0 1000G_omni2.5.hg38.vcf.gz \
  --resource:1000G,known=false,training=true,truth=false,prior=10.0 1000G_phase1.snps.high_confidence.hg38.vcf.gz \
  --resource:dbsnp,known=true,training=false,truth=false,prior=2.0  Homo_sapiens_assembly38.dbsnp138.vcf \
  -an QD -an FS -an MQ -an MQRankSum -an SOR -an ReadPosRankSum \
  -mode SNP \
  -O snps.recal --tranches-file snps.tranches \
  --rscript-file snps.plots.R

# 2. 套用 (tranche 99.7 平衡 sensitivity / precision)
gatk ApplyVQSR \
  -V cohort.vcf.gz -R GRCh38.fa \
  -mode SNP \
  --recal-file snps.recal --tranches-file snps.tranches \
  --truth-sensitivity-filter-level 99.7 \
  -O cohort.snps.vqsr.vcf.gz
# Indel VQSR 用更嚴格 tranche 99.0 (indel 較難)
gatk VariantRecalibrator \
  -V cohort.vcf.gz -R GRCh38.fa \
  --resource:mills,known=false,training=true,truth=true,prior=12.0 Mills_and_1000G_gold_standard.indels.hg38.vcf.gz \
  --resource:axiomPoly,known=false,training=true,truth=false,prior=10.0 Axiom_Exome_Plus.genotypes.all_populations.poly.hg38.vcf.gz \
  --resource:dbsnp,known=true,training=false,truth=false,prior=2.0 dbsnp.vcf.gz \
  -an QD -an FS -an MQRankSum -an SOR -an ReadPosRankSum \
  -mode INDEL \
  --max-gaussians 4   # indel 變異數較少 → 用 4 而非 8 高斯 \
  -O indels.recal --tranches-file indels.tranches

gatk ApplyVQSR \
  -V cohort.snps.vqsr.vcf.gz -R GRCh38.fa \
  -mode INDEL \
  --recal-file indels.recal --tranches-file indels.tranches \
  --truth-sensitivity-filter-level 99.0 \
  -O cohort.vqsr.vcf.gz
# GATK CNNScoreVariants (1D CNN，單樣本即可)
gatk CNNScoreVariants \
  -V cohort.vcf.gz -R GRCh38.fa \
  -O cohort.cnn.vcf.gz \
  --tensor-type reference

# 套 truth-sensitivity tranche
gatk FilterVariantTranches \
  -V cohort.cnn.vcf.gz \
  --resource hapmap_3.3.hg38.vcf.gz \
  --resource Mills_and_1000G_gold_standard.indels.hg38.vcf.gz \
  --snp-tranche 99.95 \
  --indel-tranche 99.4 \
  --info-key CNN_1D \
  -O cohort.cnn.filtered.vcf.gz

# 2D CNN (含 read pileup) — 對小樣本最佳，但需 BAM
gatk CNNScoreVariants \
  -V cohort.vcf.gz -I sample.bam -R GRCh38.fa \
  --tensor-type read_tensor \
  -O cohort.cnn2d.vcf.gz
# bcftools 快速過濾 (適合臨床 panel)
bcftools view cohort.vcf.gz -f PASS \
  -i 'QUAL>=30 && INFO/DP>=10 && INFO/MQ>=40 && FORMAT/GQ>=20' \
  -Oz -o cohort.qcpass.vcf.gz

# 套常用 sample-level filter
bcftools view cohort.qcpass.vcf.gz \
  -i 'FORMAT/AD[*:1]>=3 && FORMAT/VAF>=0.2' \
  -Oz -o cohort.sampleflt.vcf.gz
tabix cohort.sampleflt.vcf.gz

案例

gnomAD: principled VQSR

Broad Institute · 2020 · gnomAD v2 / v3 / v4 · 730k+ exomes

案例：principled VQSR

gnomAD v2 用標準 GATK VQSR，但團隊發現高 VQSLOD 區域仍有 5–10% 的「reference artifact」（基因組 reference 本身錯誤造成的假變異）。v3 採用「principled VQSR」：

  • 強化 truth set：不只 HapMap，加入 trio 一致性高的 de novo SNV、Mendelian-consistent variants
  • 引入 random forest：除了 VQSR Gaussian mixture，再跑 random forest 二次過濾，重點放在 strand bias + position-in-read 模式
  • per-feature monitoring：每個 feature 的分布跟著版本演進監控；如果 FS 分布漂移 → 重訓練
  • 透明性：每變異附 random_forest_probability + vqsr_score 雙評分

結果：v3 cohort-level FP < 1%，遠優於 v2，並啟發了 DRAGEN 採用類似的「dual scoring」概念。教訓：機器學習過濾 ≠ 全自動；需要 expert curation + 多重證據鏈。

gnomAD v2 used standard GATK VQSR but found that even high-VQSLOD regions had 5–10% "reference artifacts" (false variants caused by reference errors). v3 adopted "principled VQSR":

  • Stronger truth set: not just HapMap — added trio-consistent de novos and Mendelian-consistent variants
  • Random forest: after VQSR Gaussian mixture, a second random forest pass — focusing on strand bias and position-in-read patterns
  • Per-feature monitoring: distributions tracked version-to-version; FS drift triggers retraining
  • Transparency: each variant carries both random_forest_probability and vqsr_score

Result: v3 cohort-level FP < 1%, far better than v2, and inspired DRAGEN's "dual scoring". Lesson: ML filtering ≠ fully automated — it needs expert curation and a chain of multiple evidences.

陷阱與 tips

陷阱

對 indel 套 SNP 的 hard filter：用 FS > 60 過濾 indel → 損失 30–50% 真 indel。indel 應用 FS > 200。
Applying SNP hard filter to indels: using FS > 60 on indels loses 30–50% of true indels. Use FS > 200 for indels.
VQSR 小樣本強行跑：cohort 10 個樣本跑 VQSR → EM 不收斂或 VQSLOD 全部 NaN。請改 Hard Filter。
Forcing VQSR on small cohorts: EM diverges or VQSLOD all NaN for cohorts of 10. Use Hard Filter instead.
FilterMutectCalls vs VQSR 混淆：somatic 是 FilterMutectCalls（含 contamination + orientation 模型），不是 VQSR。VQSR 假設 germline het ≈ 50% VAF，套在 somatic 上會把低 VAF clonal mut 全刪掉。
FilterMutectCalls vs VQSR confusion: somatic uses FilterMutectCalls (with contamination + orientation models), not VQSR. VQSR assumes germline het ≈ 50% VAF — running on somatic deletes all low-VAF clonal mutations.
只看 site-level filter 忽略 sample-level：FILTER 欄 PASS 不代表「每個樣本都 PASS」。要看 FORMAT 的 GQ、DP、AD —— GQ < 20 / DP < 10 的單樣本基因型應視為「未呼叫」(./.)。
Site-level filter overrides sample-level: a PASS in FILTER doesn't mean every sample passes. Inspect FORMAT GQ, DP, AD — per-sample genotypes with GQ < 20 or DP < 10 should be set to "./." (no-call).
QC dashboard：用 bcftools stats + plot-vcfstats + multiqc 看 Ti/Tv、het/hom ratio、indel size distribution → 一頁 HTML 報告，快速判讀 cohort 健康。
QC dashboard: use bcftools stats + plot-vcfstats + multiqc to view Ti/Tv, het/hom ratio, indel size distribution → one-page HTML report for cohort health.
hap.py 評估：有 GIAB truth set 的話用 hap.py truth.vcf query.vcf -f benchmark.bed 跑 precision / recall / F1，輸出 ROC 曲線。是業界標配。
Evaluate with hap.py: with GIAB truth set, run hap.py truth.vcf query.vcf -f benchmark.bed for precision / recall / F1 with ROC curves. Industry standard.

Transitions (A↔G, C↔T) are biologically more frequent than transversions (CpG deamination), so real human WGS Ti/Tv ≈ 2.0–2.1; WES CDS, CpG-rich, ~3.0–3.3. FPs are random (Ti = Tv, ratio ~0.5). Cohort Ti/Tv at 1.5 → many FPs; 2.5 (WGS) → over-filtered. Free sanity check.
兩個原因：(1) truth set 不平衡 — 罕見族群的真變異不在 HapMap，落在 cluster 外被視為 novel；(2) tranche 設太嚴 (99.0) — 把 sensitivity 降到 99.0% 意味著刪 0.7% truth 加大量 novel。臨床建議：highly-curated panel 不跑 VQSR，直接用 hard filter + 雙 caller 交集。
Two reasons: (1) imbalanced truth — rare-population real variants not in HapMap fall outside the cluster and look novel; (2) tranche too strict (99.0) sacrifices 0.7% truth alongside novel removal. Clinical advice: don't run VQSR on highly-curated panels; use hard filter + dual-caller intersection.
