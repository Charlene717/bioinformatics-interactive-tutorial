為什麼重要

預處理是準確度的天花板

Variant calling 的 false positive 有 70% 以上來自預處理瑕疵——錯誤的 mapping、未標記的 PCR duplicates、過度自信的 base quality。如果這個階段做不好，後面再強的 caller (DeepVariant、Mutect2) 也救不回來：「Garbage in, garbage out」在 NGS 領域成立得最嚴格。

本章將帶你看清三個關鍵步驟——BWA-MEMBurrows-Wheeler Aligner with MEM (maximal exact match) seeding；2009 年由 Heng Li 發表，至今仍是短讀比對的事實標準。、MarkDuplicatesPCR 與 optical duplicates 的標記算法；不真正刪除 reads，而是設定 BAM flag 1024 (0x400)。、BQSRBase Quality Score Recalibration；用機器學習糾正鹼基品質的系統性偏差。——背後的原理、陷阱與最佳實踐。

Over 70% of variant-calling false positives come from preprocessing flaws — incorrect mapping, unmarked PCR duplicates, overconfident base qualities. Get this stage wrong and no caller (DeepVariant, Mutect2) can save you: "Garbage in, garbage out" applies most strictly in NGS.

This chapter walks through three crucial steps — BWA-MEMBurrows-Wheeler Aligner with MEM (maximal exact match) seeding; published in 2009 by Heng Li, still the de-facto standard for short-read alignment., MarkDuplicatesAlgorithm to mark PCR and optical duplicates; does not delete reads but sets BAM flag 1024 (0x400)., BQSRBase Quality Score Recalibration; ML-based correction of systematic base-quality bias. — and the principles, pitfalls, and best practices behind each.

三大步驟

預處理三步曲

比對 (BWA-MEM)

BWA-MEM 用 Burrows-Wheeler transform (BWT)一種無損文字壓縮 + 索引技術，讓 read 在 4 GB 基因組中的 exact match 查找變為 O(read_length)。 + suffix array字串所有 suffix 的排序索引，用來找 alignment seed。建索引，用 maximal exact match (MEM) 作為 seed，再用 Smith-Waterman 算分。

BWA-MEM uses Burrows-Wheeler transform (BWT)A lossless text compression + indexing technique enabling O(read_length) exact-match lookups in a 4 GB genome. + suffix arrayA sorted index of all string suffixes, used as alignment seeds. for indexing; finds maximal exact matches as seeds, then Smith-Waterman scoring.

標記重複

PCR/optical 重複是 false positive 的主要來源。MarkDuplicates 用「同 5' 端座標 + 同方向」判定為重複，保留 base-quality 加總最高者。

PCR/optical duplicates are major FP sources. MarkDuplicates identifies duplicates by "same 5' coordinate + same orientation," keeping the one with highest summed base quality.

BQSR

Phred 30 應該對應 1/1000 錯誤率，但實際機器偏差讓「Q30」可能是 Q25 或 Q35。BQSR 用 dbSNP 已知變異當「不會錯」訓練集，校正每種 (cycle, context, quality) 組合的偏差。

A reported Phred 30 should mean 1/1000 error rate, but instrument bias makes "Q30" actually Q25 or Q35. BQSR uses dbSNP variants as a "ground truth" training set to recalibrate per (cycle, context, quality) bin.

BWA-MEM 實作

# 2. 加 ReadGroup 並比對；-M 標記次要比對 (供 Picard 相容)
bwa mem -t 16 -M \
  -R "@RG\tID:HG002_L001\tSM:HG002\tLB:lib_HG002_1\tPL:ILLUMINA\tPU:HVT5LBBXX.1.HG002" \
  GRCh38 HG002_R1.fq.gz HG002_R2.fq.gz \
  | samtools sort -@ 8 -O bam -o HG002.aligned.bam -

samtools index HG002.aligned.bam
samtools flagstat HG002.aligned.bam   # 檢查 mapping rate
# PacBio HiFi (CCS) variant calling preset
minimap2 -ax map-hifi -t 16 \
  -R "@RG\tID:HG002_HiFi\tSM:HG002\tPL:PACBIO\tLB:hifi" \
  GRCh38.mmi HG002.hifi.fq.gz \
  | samtools sort -@ 8 -o HG002.hifi.bam -

# Oxford Nanopore Q20+ reads
minimap2 -ax map-ont -t 16 GRCh38.mmi ont.fq.gz | samtools sort -o ont.bam -

ReadGroup 解剖

@RG 解剖學

許多 GATK 工具沒看到 RG 就直接報錯。這不是刁難——RG 把 read 跟「樣本身份、文庫、平台、批次」綁定，是下游 sample-level QC、batch correction、cross-sample contamination 偵測的命脈。

Many GATK tools refuse to run without RG. This isn't pedantry — RG ties reads to sample identity, library, platform, and batch, which is the foundation of sample-level QC, batch correction, and cross-sample contamination detection.

Wrong ReadGroup = downstream 災難：常見錯誤是 50 個樣本通通寫 SM:sample，joint genotyping 出來只剩一個樣本欄位。或同樣本多 lane 用同一 ID → GATK 報 duplicate readgroup。
Wrong ReadGroup = downstream disaster: common error — 50 samples all share SM:sample, joint genotyping yields just one sample column. Or same sample's lanes share one ID → GATK errors duplicate readgroup.
實用約定：多 lane 同樣本 → 共 SM、共 LB（若同 library 多 lane），每 lane 一個獨立 ID 與 PU。BQSR 才能對每個 lane 學各自的 error model。
Useful convention: for multi-lane samples → share SM, share LB (if same library), unique ID and PU per lane. This lets BQSR learn per-lane error models.
MarkDuplicates

為什麼要標記重複？

PCR 把同一個 DNA 片段擴增多份，定序到的「多個 read」其實源自同一原始分子。若不標記，計算 allele depth 時會把同一突變算多次 → 過度自信、假陽性。

常見 duplicate rate：WGS 5–15% (PCR-free 可 < 5%)；WES 因 capture step PCR 多達 20–40%；UMI-based panel 可降至 < 1%。

PCR amplifies the same fragment many-fold; the resulting "reads" originate from one molecule. Without marking, allele depth double-counts the same mutation → over-confidence and false positives.

Typical rates: WGS 5–15% (PCR-free < 5%); WES 20–40% (capture PCR); UMI-based panel < 1%.

MarkDuplicates 區分兩類重複：

  • PCR duplicates：同 5' coordinate + 同 orientation + 同 LB → 標記。
  • Optical duplicates：同 flowcell 上座標距離 < 100 pixels (HiSeq/NovaSeq)，源自單一 cluster 被誤判為多個 → 另外標記、不影響 dedup。

保留策略：在所有 duplicates 中，留 base-quality 加總最高者，其他打上 BAM flag 1024 (0x400)。

MarkDuplicates separates two classes:

  • PCR duplicates: same 5' coordinate + orientation + LB → marked.
  • Optical duplicates: within 100 px on flowcell (HiSeq/NovaSeq); one cluster mis-recognized as multiple — flagged separately.

Keeper: in each duplicate set, retain the read with highest summed base quality; flag others with BAM bit 1024 (0x400).

數學上，若一個位點被 PCR 擴增 N 次、每次正確讀到變異 read 的機率 p，未 dedup 時 alt-depth 估計：

E[alt_depth] = N · p
Var[alt_depth] = N · p(1−p)
但「真實獨立分子數」k = 1（PCR 同源）Without dedup → effective sample size = N, but true independent = 1

結果：variance 被低估 N 倍，statistical confidence (Phred Q) 被高估，FP 暴增。Dedup 後 N → 1，恢復正確 binomial。

Mathematically, if a site is PCR-amplified N times with per-amp variant-read probability p, the un-dedup'd alt-depth estimator:

E[alt_depth] = N · p
Var[alt_depth] = N · p(1−p)
but true independent k = 1 (same PCR origin)Effective n = N pre-dedup; true independent = 1

Variance is under-estimated N-fold, statistical confidence (Phred Q) inflated, FPs explode. After dedup N → 1, correct binomial is restored.

BQSR 深入

BQSR 統計直覺

定序機回報的 Phred 分數Q = −10 log10(P_error)。Q30 ≈ 1/1000 錯、Q40 ≈ 1/10000 錯。與「真實錯誤率」之間有系統性偏差：cycle 後期錯誤率上升、特定 context (CpG, G>T) 有 sequencing-specific bias、機器型號不同也影響。

BQSR 的核心假設：「已知 SNP (dbSNP) 上的『錯誤』，幾乎都不是真的錯誤」。因此可在 non-known-site 上把所有 mismatch 視為錯誤，學習 (cycle × context × reported_Q) 三維 bin 的實際錯誤率，再校正每個 base 的 Phred score。

The reported Phred scoreQ = −10 log10(P_error). Q30 ≈ 1/1000 error, Q40 ≈ 1/10000. vs true error rate has systematic bias: late-cycle errors rise, certain contexts (CpG, G>T) have machine-specific bias, instrument model matters.

BQSR's core assumption: "errors" landing on known SNPs (dbSNP) are mostly not real errors. So at non-known sites, all mismatches are treated as errors. The algorithm learns actual error rate per (cycle × context × reported_Q) bin and recalibrates each base's Phred.

Qnew = −10 log10( #errors(cycle, ctx, Q) / #total(cycle, ctx, Q) ) BQSR per-bin empirical recalibration
把 BQSR 想成體重計校正：你買了新體重計，要先放幾個已知重量的物體（5 kg 啞鈴、10 kg 啞鈴）來「校零」。BQSR 中的 dbSNP 就是這些已知物體；每個（位置、特徵）組合都會學到一個系統性偏移量。
BQSR is like calibrating a new scale: you place known-weight items (5 kg dumbbell, 10 kg dumbbell) to "zero" the scale. dbSNP is BQSR's known weights; each (cycle, context) combination learns a systematic offset.
BQSR without known sites：非模式生物（沒有 dbSNP）跑 BQSR 會把所有 mismatch 視為錯誤、嚴重過度校正，base quality 集體被壓低。解法：bootstrap BQSR（先呼叫一輪變異 → 高信心變異當 known sites → 重跑 BQSR），或直接跳過 BQSR。
BQSR without known sites: non-model species (no dbSNP) treat all mismatches as errors, causing severe over-correction; base qualities are uniformly suppressed. Fix: bootstrap BQSR (call variants → use high-confidence calls as known → re-run BQSR), or skip BQSR.
BQSR 完後 quality 看起來變差：很多人第一次跑 BQSR 看到 mean Q 從 35 降到 32 會懷疑做錯。其實這是正確的——機器一開始就高估了。校正後雖然平均較低，但更準。
"BQSR made my quality worse": many first-time users see mean Q drop from 35 → 32 and worry. This is correct: the instrument over-reported. Post-BQSR is lower on average but more accurate.

BQSR 實作

# Step 1: 建立 recalibration table
gatk BaseRecalibrator \
  -I HG002.dedup.bam -R GRCh38.fa \
  --known-sites Homo_sapiens_assembly38.dbsnp138.vcf \
  --known-sites Mills_and_1000G_gold_standard.indels.hg38.vcf.gz \
  --known-sites Homo_sapiens_assembly38.known_indels.vcf.gz \
  -O HG002.recal_table.txt

# Step 2: 套用校正
gatk ApplyBQSR \
  -I HG002.dedup.bam -R GRCh38.fa \
  --bqsr-recal-file HG002.recal_table.txt \
  -O HG002.analysis_ready.bam
# 非模式物種 bootstrap BQSR（無 dbSNP）
# Round 1：先用最寬鬆條件 call 一輪
gatk HaplotypeCaller -I dedup.bam -R ref.fa -O round1.vcf.gz

# Round 2：篩高品質變異當「pseudo-known sites」
gatk VariantFiltration -V round1.vcf.gz \
  --filter-expression "QD < 10 || FS > 60 || MQ < 50" \
  --filter-name "lowQ" -O round1.flt.vcf.gz
bcftools view -f PASS round1.flt.vcf.gz -Oz -o pseudo_known.vcf.gz
tabix pseudo_known.vcf.gz

# Round 3：用 pseudo_known 跑 BQSR
gatk BaseRecalibrator -I dedup.bam --known-sites pseudo_known.vcf.gz -O recal.tbl
gatk ApplyBQSR -I dedup.bam --bqsr-recal-file recal.tbl -O analysis_ready.bam
# 看校正前後對比（before vs after table）
gatk BaseRecalibrator -I analysis_ready.bam -R GRCh38.fa \
  --known-sites dbsnp.vcf.gz -O after_recal.txt

gatk AnalyzeCovariates \
  -before HG002.recal_table.txt \
  -after  after_recal.txt \

案例

GIAB HG002 案例

NIST · Genome in a Bottle Consortium · 2014–present · HG002 / NA24385

案例：HG002 truth set 是如何建構的？

HG002 是來自 Personal Genome Project 的 Ashkenazi Jewish 父子三人組 (HG002 son, HG003 father, HG004 mother)。NIST 用同一檢體在 5 個定序平台 (Illumina, PacBio, ONT, 10X Linked-Reads, BioNano)、6 種 caller、3 個 reference去比對，取共識作為 high-confidence truth set。

關鍵教訓：(1) 沒有任何單一 caller 或單一平台能達到 100% 召回——交集才是黃金；(2) high-confidence region 只覆蓋約 88% 基因組——其餘 12% 含 segmental duplications、centromeres、telomeres，是 "dark matter"；(3) 評估自己的 pipeline 時，要 hap.py + GIAB BED 子集 比較，否則被「reference 無法評估區」拖累 precision。

HG002 啟發了 PrecisionFDA Truth Challenge——DeepVariant、DRAGEN、Strelka2 等工具的精準度競賽。2022 年最佳 SNV F1 已 > 99.95%。

HG002 is the son of an Ashkenazi Jewish trio (HG002 son, HG003 father, HG004 mother) from the Personal Genome Project. NIST sequenced the same biospecimen on 5 platforms (Illumina, PacBio, ONT, 10X, BioNano), 6 callers, 3 references, and used consensus as the high-confidence truth set.

Key lessons: (1) no single caller or platform reaches 100% recall — intersection is gold; (2) high-confidence regions cover only ~88% of the genome — the remaining 12% (segdups, centromeres, telomeres) are "dark matter"; (3) evaluate your pipeline with hap.py restricted to the GIAB BED, else "un-evaluatable regions" tank your precision.

HG002 inspired the PrecisionFDA Truth Challenge — a precision contest among DeepVariant, DRAGEN, Strelka2, etc. As of 2022, best SNV F1 > 99.95%.

常見問題

常見問題

人類 + Illumina：不建議跳過，會多 5–10% FP。但 DeepVariant 內建 CNN 不太依賴 base quality，可省略 BQSR (作者實測 F1 影響 < 0.1%)。非人類無 dbSNP：建議跳過或 bootstrap。
Human + Illumina: do not skip; 5–10% extra FPs. But DeepVariant's CNN doesn't rely on base quality much (author benchmarks: ΔF1 < 0.1%). Non-human, no dbSNP: skip or bootstrap.
只標記 (REMOVE_DUPLICATES=false)，保留檔案完整性。所有下游 caller 預設會自動忽略 flag 1024 的 reads。例外：超大檔案儲存空間吃緊時可移除以省 10–20%。
Just flag (REMOVE_DUPLICATES=false) to keep file integrity. All callers ignore flag 1024 by default. Exception: tight storage on huge cohorts — removing saves 10–20%.
BWA-MEM 多執行緒下對「multi-mapper」會隨機選一個（同分隨機）。要 deterministic 可加 -K + 單執行緒，或設 seed。實際對 SNV calling 結果影響 < 0.01% (除非高度重複區)。
BWA-MEM in multi-thread mode randomly picks one alignment for multi-mappers (tie-breaking). For determinism, use -K + single thread, or fix the seed. SNV-calling impact < 0.01% (except in highly repetitive regions).
WES 多了 hybridization capture 步驟，每個 probe 抓固定區域後再 PCR 多放大 → library complexity 偏低。WGS PCR-free protocol 可達 < 5% dup；WES 即使優化也很難低於 15–20%。
WES has an extra hybridization capture step — each probe targets a fixed region then PCR amplifies, lowering library complexity. WGS PCR-free can reach < 5% dup; WES rarely drops below 15–20% even optimized.
GRCh38 含 261 個 ALT 序列，代表 HLA / MHC 等高多態區的另一條 haplotype。BWA-MEM 預設「ALT-aware」會把 reads 散到主鏈與 ALT 上，下游 caller 預設只看 primary chr → 不會錯。不要移除 ALT，但要用 .alt 索引啟用 ALT-aware mapping。
GRCh38 has 261 ALT contigs — alternative haplotypes for HLA/MHC etc. ALT-aware BWA-MEM distributes reads to primary + ALT; downstream callers see primary only. Don't remove ALTs — use the .alt index to enable ALT-aware mapping.
用 -m 4G -@ 8 -T tmp/：設每執行緒記憶體上限 4 GB + 中介檔案目錄。samtools 會自動分塊排序、最後 merge。WGS 30× BAM 通常需 ~80 GB temp space。
Use -m 4G -@ 8 -T tmp/ — cap per-thread RAM at 4 GB + give a tmp dir. samtools chunks and merges automatically. WGS 30× BAM typically needs ~80 GB temp.
