Read 預處理

BWA-MEM 比對、MarkDuplicates、BQSR——把原始 reads 變成「分析就緒」的 BAM。

BWA-MEM, MarkDuplicates, BQSR — turn raw reads into an "analysis-ready" BAM.

三大步驟

預處理三步曲

🎯

1. 比對

BWA-MEM 是 70 bp+ short read 對人類基因組的事實標準。輸出 BAM 含座標。

BWA-MEM is the de facto standard for 70 bp+ short reads on human genome. Outputs coordinate-sorted BAM.

🔁

2. 標記重複

PCR / optical 重複是 false positive 的主要來源。用 picard MarkDuplicates 或 GATK4 MarkDuplicatesSpark。

PCR / optical duplicates are a major false-positive source. Use picard MarkDuplicates or GATK4 MarkDuplicatesSpark.

📏

3. BQSR

BaseRecalibrator + ApplyBQSR：用已知 SNP 資料庫，學習機器與化學造成的鹼基品質偏差並重新校正。

BaseRecalibrator + ApplyBQSR: learn machine + chemistry bias from known SNPs, then recalibrate base qualities.

比對策略

BWA-MEM 用法要點

# 1. 建索引（一次性，約 30 GB 記憶體）
bwa index -p GRCh38 -a bwtsw GRCh38.fa

# 2. 加 ReadGroup 並比對（最重要：RG！）
bwa mem -t 16 -M \
  -R "@RG\tID:lib1\tSM:patient_01\tLB:lib1\tPL:ILLUMINA" \
  GRCh38 R1.fq.gz R2.fq.gz \
  | samtools sort -@ 8 -o aligned.bam -

samtools index aligned.bam
⚠️
ReadGroup (RG) 是必填：下游 GATK 工具讀 RG 識別樣本身份。漏掉 RG 會在 HaplotypeCaller 出錯。最少需 ID, SM, LB, PL。
ReadGroup (RG) is mandatory: GATK tools read RG to identify samples. Missing RG breaks HaplotypeCaller. Minimum: ID, SM, LB, PL.
MarkDuplicates

為什麼要標記重複？

PCR 把同一個 DNA 片段擴增多份，定序到的「多個 read」其實源自同一原始分子。若不標記、計算 allele depth 時會把同一突變算多次 → 過度自信、假陽性。

PCR amplifies the same fragment many-fold; the resulting "reads" come from one original molecule. Without marking, allele depth double-counts the same mutation → over-confidence and false positives.

# Picard 版（穩定）
picard MarkDuplicates \
  I=aligned.bam \
  O=dedup.bam \
  M=dedup_metrics.txt \
  REMOVE_DUPLICATES=false   # 只標記不移除（推薦）

samtools index dedup.bam
BQSR

鹼基品質再校正

定序機回報的 Phred 分數與「真實錯誤率」之間有系統性偏差（受循環次數、context、機器型號影響）。BQSR 利用已知 SNP（如 dbSNP）作為「不會錯」的訓練集，學習偏差並校正其他位置的分數。

Reported Phred quality vs true error rate has systematic bias (cycle, context, instrument). BQSR uses known SNPs (dbSNP) as a "ground truth" training set, learns the bias, and recalibrates everywhere else.

# Step 1: 建立 recalibration table
gatk BaseRecalibrator \
  -I dedup.bam -R GRCh38.fa \
  --known-sites dbsnp.vcf.gz \
  --known-sites Mills_indels.vcf.gz \
  -O recal_table.txt

# Step 2: 套用校正
gatk ApplyBQSR \
  -I dedup.bam -R GRCh38.fa \
  --bqsr-recal-file recal_table.txt \
  -O analysis_ready.bam
