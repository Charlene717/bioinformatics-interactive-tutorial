SV / CNV 偵測

大片段變異——SNV/Indel 之外的另一個世界。

Large-scale variants — another world beyond SNV/Indel.

核心概念

SV vs CNV

結構變異 (Structural Variant, SV) 涵蓋 > 50 bp 的大片段變化，包括 deletion (DEL)、duplication (DUP)、insertion (INS)、inversion (INV)、translocation (TRA)、breakend (BND)。拷貝數變異 (CNV) 是 SV 的子集，特指 DEL 與 DUP，數量變化以 kb 至 Mb 計。

Structural variants (SV) cover > 50 bp changes: deletion (DEL), duplication (DUP), insertion (INS), inversion (INV), translocation (TRA), breakend (BND). Copy-number variants (CNV) are a subset — DEL and DUP at kb to Mb scale.

偵測訊號

四種訊號

1. Read depth (RD)

DUP 區域 reads 翻倍、DEL 區域 reads 減半——對 CNV 最直觀，需大量定序。工具：CNVkit、Control-FREEC、GATK gCNV。

DUP doubles depth, DEL halves it — most intuitive for CNV; needs decent depth. Tools: CNVkit, Control-FREEC, GATK gCNV.

2. Read pair (RP)

paired-end 之間距離若超出預期（或方向倒置），暗示有 DEL / INV。

If paired-end distance/orientation deviates from expected, suggests DEL / INV.

3. Split read (SR)

read 一半比對 chr1、一半比對 chr5 → translocation 或 INS 的標記。

Half a read aligns to chr1 and half to chr5 → translocation or insertion marker.

4. De novo assembly

無法比對的 reads 組裝後與基因組比對，找到 novel insertion / inversion。

Unmapped reads assembled then re-aligned — finds novel insertions / inversions.

💡
長讀定序 (PacBio HiFi、ONT) 的優勢：單條 read 跨越大部分 SV 與重複序列，靈敏度遠勝 short reads。短讀對 < 1 kb SV 漏報率高。
Long-read advantage (PacBio HiFi, ONT): single reads span most SVs and repeats. Short reads miss many < 1 kb SVs.

程式碼

實作範例

# Manta germline
configManta.py \
  --bam analysis_ready.bam \
  --referenceFasta GRCh38.fa \
  --runDir manta_run/

manta_run/runWorkflow.py -j 8
# 輸出在 manta_run/results/variants/diploidSV.vcf.gz
# 1. 計算 read counts
gatk CollectReadCounts \
  -I sample.bam -L exons.preprocessed.interval_list \
  --interval-merging-rule OVERLAPPING_ONLY \
  -O sample.counts.hdf5

# 2. 群體式呼叫 CNV (cohort mode)
gatk GermlineCNVCaller \
  --input sample1.counts.hdf5 --input sample2.counts.hdf5 ... \
  --contig-ploidy-calls ploidy_calls/ \
  --output cnv_out/ --output-prefix cohort \
  --run-mode COHORT
cnvkit.py batch \
  tumor.bam --normal normal.bam \
  --targets exons.bed --annotate refFlat.txt \
  --fasta GRCh38.fa \
  --output-reference my_reference.cnn \
  --output-dir cnvkit_out/

# 視覺化
cnvkit.py scatter -s sample.cns sample.cnr -o sample.scatter.png
cnvkit.py diagram  -s sample.cns sample.cnr -o sample.diagram.pdf
陷阱

常見錯誤

🚫
WES 樣本跑 WGS-only 工具：WES 在 introns 沒覆蓋，許多 SV 工具會錯誤呼叫。WES 適合 GATK gCNV、CNVkit；別用 Manta 對 WES 跑 SV。
WES + WGS-only tools: WES lacks intronic coverage; many SV tools mis-call. Use GATK gCNV / CNVkit for WES; don't run Manta SV on WES.
🚫
只用單一工具：每個 SV 工具靈敏度與精準度互補。建議跑 Manta + Delly + GRIDSS，取交集或用 SURVIVOR 合併。
Single-tool reliance: SV callers have complementary sensitivity/precision. Run Manta + Delly + GRIDSS; intersect or merge with SURVIVOR.
🚫
沒過濾低品質區：centromere、telomere、segmental duplication 是 SV 假陽性的溫床——用 ENCODE blacklist + segmental dup region 過濾。
Skipping low-quality regions: centromeres, telomeres, segmental dups are SV FP hotspots — filter with ENCODE blacklist + segmental dup tracks.
