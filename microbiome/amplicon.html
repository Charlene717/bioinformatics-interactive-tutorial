16S / ITS Amplicon

最普及的微生物群體調查方式——用一段保守基因當條碼識別細菌或真菌。

The most popular microbiome survey — use a conserved gene as a barcode for bacteria or fungi.

基本概念

16S rRNA 結構

16S rRNA 是所有細菌都有、約 1500 bp 的核醣體 RNA 基因。它有 9 個可變區 (V1-V9) 與保守區交替。常見策略是只 PCR 一兩個可變區：

16S rRNA is a ~1500 bp ribosomal RNA gene present in all bacteria, with 9 variable regions (V1-V9) interleaved with conserved regions. Common strategy: PCR-amplify 1-2 variable regions:

OTU vs ASV

OTU vs ASV

🟡

OTU (Operational Taxonomic Unit)

傳統做法：把相似度 ≥ 97% 的序列聚類成一個 OTU。優：耐受定序錯誤；限：丟失子物種精度，且 OTU 邊界不穩定。

Traditional: cluster sequences ≥ 97% similarity into one OTU. Pro: tolerates seq errors. Con: loses sub-species detail; OTU boundaries unstable.

🟢

ASV (Amplicon Sequence Variant)

2017 後主流：用 denoising 模型 (DADA2、Deblur) 把錯誤從序列「校正回真實」，每個唯一序列即一個 ASV。優：解析到單鹼基；可跨研究比較。

Post-2017 standard: denoising models (DADA2, Deblur) "correct errors back to truth"; each unique sequence = one ASV. Pro: single-base resolution; cross-study comparable.

💡
當代建議：新分析一律用 ASV (DADA2)。OTU 主要用於回溯老資料或計算量考量。
Modern recommendation: Use ASV (DADA2) for new analyses. OTU is mainly for legacy data or compute constraints.

主流流程

DADA2 (R)QIIME2mothurvsearchPICRUSt2
library(dada2)
path <- "raw_fastqs/"
fnF <- sort(list.files(path, pattern = "_R1.fastq.gz"))
fnR <- sort(list.files(path, pattern = "_R2.fastq.gz"))
filtF <- file.path("filt", basename(fnF))
filtR <- file.path("filt", basename(fnR))

# 1. 品質過濾
filterAndTrim(fnF, filtF, fnR, filtR,
              truncLen   = c(240, 200),
              maxEE      = c(2, 2),
              truncQ     = 2,
              rm.phix    = TRUE, multithread = TRUE)

# 2. 估計錯誤率 + 主流程
errF <- learnErrors(filtF, multithread = TRUE)
errR <- learnErrors(filtR, multithread = TRUE)
ddF  <- dada(filtF, err = errF, multithread = TRUE)
ddR  <- dada(filtR, err = errR, multithread = TRUE)
mrg  <- mergePairs(ddF, filtF, ddR, filtR)
seqtab <- makeSequenceTable(mrg)

# 3. 去 chimera + 分類
seqtab.nc <- removeBimeraDenovo(seqtab, method = "consensus")
tax <- assignTaxonomy(seqtab.nc, "silva_nr99_v138_train_set.fa.gz")
# QIIME2 完整流程
qiime tools import --type 'SampleData[PairedEndSequencesWithQuality]' \
  --input-path manifest.csv \
  --input-format PairedEndFastqManifestPhred33V2 \
  --output-path demux.qza

qiime dada2 denoise-paired \
  --i-demultiplexed-seqs demux.qza \
  --p-trunc-len-f 240 --p-trunc-len-r 200 \
  --p-n-threads 8 \
  --o-table table.qza --o-representative-sequences rep-seqs.qza --o-denoising-stats stats.qza

qiime feature-classifier classify-sklearn \
  --i-classifier silva-138-99-nb-classifier.qza \
  --i-reads rep-seqs.qza \
  --o-classification taxonomy.qza

下游分析

核心下游分析

Alpha / Beta diversity

用 phyloseq、vegan、scikit-bio。Alpha 比較組間多樣性；Beta + PERMANOVA 比較群落結構差異。

phyloseq, vegan, scikit-bio. Alpha compares diversity across groups; Beta + PERMANOVA tests composition shifts.

差異豐度

微生物比例資料專用：ALDEx2、ANCOM-BC、MaAsLin2、LEfSe。避免直接用 t-test。

Compositional-aware tools: ALDEx2, ANCOM-BC, MaAsLin2, LEfSe. Avoid t-test on raw proportions.

物種網絡

SparCC、SPIEC-EASI 建立物種共生網絡，找微生物社群結構。

SparCC, SPIEC-EASI build co-occurrence networks to reveal microbial communities.
