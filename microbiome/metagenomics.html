Random sequencing of the whole community — get strain-level composition and the full functional gene catalog.

兩大策略

Read-based vs Assembly-based

📖

Read-based

把每條 read 直接對照已知基因組 / 蛋白資料庫分類。優：快、低門檻。限：依賴 reference；無法找新物種。

Map each read to known genomes / proteins. Pro: fast, low effort. Con: reference-dependent; misses novel organisms.

工具：Kraken2、Bracken、MetaPhlAn4、Centrifuge、KrakenUniq。

Tools: Kraken2, Bracken, MetaPhlAn4, Centrifuge, KrakenUniq.

🧩

Assembly-based

先 de novo 組裝 → 把 contigs 分箱 (binning) → 還原可能的 MAG (metagenome-assembled genome)。

De novo assemble → bin contigs → reconstruct MAGs (metagenome-assembled genomes).

工具：metaSPAdes、MEGAHIT、MetaBAT2、CONCOCT、CheckM2、GTDB-Tk。

Tools: metaSPAdes, MEGAHIT, MetaBAT2, CONCOCT, CheckM2, GTDB-Tk.

Read-based 流程

Kraken2 + Bracken

Kraken2 使用 k-mer 精確匹配快速分類，速度 8 GB/min；Bracken 用 Bayesian 估算物種豐度。MetaPhlAn4 走另一條路——只比對 clade-specific marker genes，速度與準確度兼具。

Kraken2 uses exact k-mer matching at 8 GB/min; Bracken adds Bayesian abundance. MetaPhlAn4 takes another path — aligning only to clade-specific marker genes for balanced speed/accuracy.

# 1. Kraken2 分類
kraken2 --db kraken2_db \
  --paired R1.fq.gz R2.fq.gz \
  --threads 16 --gzip-compressed \
  --report sample1.k2.report \
  --output sample1.k2.out

# 2. Bracken 物種層級豐度
bracken -d kraken2_db \
  -i sample1.k2.report \
  -o sample1.bracken \
  -r 150 -l S
# MetaPhlAn4：直接給物種與豐度
metaphlan R1.fq.gz,R2.fq.gz \
  --bowtie2db metaphlan4_db \
  --input_type fastq \
  --bowtie2out sample1.bt2.bz2 \
  --nproc 12 \
  -o sample1.metaphlan4.tsv

# 合併多樣本
merge_metaphlan_tables.py *.metaphlan4.tsv > merged_abundance.tsv
Assembly 流程

MAG 重建完整流程

Trim→ Assembly→ Mapping→ Binning→ Refinement→ CheckM2→ GTDB-Tk 
# MEGAHIT 對大資料快
megahit -1 trim_R1.fq.gz -2 trim_R2.fq.gz \
  -o megahit_out --min-contig-len 1500 -t 16
# 1. 映射回 contigs 估深度
bowtie2-build megahit_out/final.contigs.fa contigs_idx
bowtie2 -x contigs_idx -1 trim_R1.fq.gz -2 trim_R2.fq.gz \
  | samtools sort -@ 8 -o mapping.bam -
jgi_summarize_bam_contig_depths --outputDepth depth.txt mapping.bam

# 2. MetaBAT2 分箱
metabat2 -i megahit_out/final.contigs.fa \
  -a depth.txt -o bins/bin -m 1500 -t 16
# 評估 MAG 完整度與汙染
checkm2 predict --threads 16 --input bins/ \
  --output-directory checkm2_out

# GTDB-Tk 分類
gtdbtk classify_wf \
  --genome_dir bins/ \
  --out_dir gtdbtk_out --extension fa \
  --cpus 16
