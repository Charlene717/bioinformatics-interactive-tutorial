核心概念

兩條路：shotgun vs 16S

📚

Shotgun: 直接

直接從 reads 比對到蛋白家族（UniRef、KEGG Orthology）。實際存在的功能基因，最精準。

Map reads directly to protein families (UniRef, KEGG Orthology). Actual genes — most accurate.

工具：HUMAnN3、SqueezeMeta、MetaQUAST。

Tools: HUMAnN3, SqueezeMeta, MetaQUAST.

🔮

16S: 推估

用已知物種的「基因組功能輪廓」推估 16S 樣本可能有什麼基因。限：誤差大，不能取代 shotgun。

Predict gene content of a 16S sample from known reference genomes. Caveat: uncertain — not a shotgun substitute.

工具：PICRUSt2、Tax4Fun2、Piphillin。

Tools: PICRUSt2, Tax4Fun2, Piphillin.

陷阱

常見錯誤

🚫
把 16S 預測當實驗事實：PICRUSt2 只是「推估」，準確度依賴參考樹涵蓋度。談機制前一定要 shotgun 驗證。
Treating 16S predictions as ground truth: PICRUSt2 only predicts; accuracy depends on tree coverage. Validate with shotgun before mechanistic claims.
🚫
用 raw counts 做 pathway 比較：樣本間 library size 差異會干擾——先 CPM / CLR 正規化。
Pathway compare on raw counts: library-size differences confound — normalize with CPM / CLR.
🚫
忽略 host DNA：人類腸道樣本可能含 30%+ 人類 DNA，會稀釋微生物訊號。先用 Bowtie2 對人類基因組過濾。
Ignoring host DNA: human gut samples may contain 30%+ human DNA diluting microbial signal. Pre-filter with Bowtie2 against host genome.
