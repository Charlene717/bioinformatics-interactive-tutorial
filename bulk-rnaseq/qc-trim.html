原始品質管控與修剪

在比對之前先檢視 reads 品質、移除接頭與低品質鹼基——garbage in, garbage out。

Inspect read quality and remove adapters / low-quality bases before alignment — garbage in, garbage out.

檔案格式

認識 FASTQ

FASTQ 是定序儀的標準輸出格式，每筆 read 佔 4 行：

FASTQ is the standard sequencer output. Each read takes 4 lines:

@SRR1039508.1 HWI-ST177:290:C0TECACXX:1:1101:1225:2130 length=63
GAACAGCGGCTTGGGCAGCGAGCAGCTGCACCCTGAGGGGCCACTTTCAGGCAACTTCCATG
+
CCCFFFFFHHHHHJJIIJIJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJIJJJIJJJIJJJ

Line 1

@ 開頭，後面是 read ID 與機器資訊。

@ followed by read ID and sequencer metadata.

Line 2

DNA 序列 (A/C/G/T/N)，N 表示無法判讀。

DNA sequence (A/C/G/T/N); N marks ambiguous calls.

Line 3

+ 分隔符。

A + separator.

Line 4

Phred 品質分數（ASCII 編碼，Q33）。Q30 代表錯誤率 0.1%。

Phred quality (ASCII Q33). Q30 means 0.1% error rate.

💡
Phred 公式：Q = -10·log10(P_error)。Q20 = 1%、Q30 = 0.1%、Q40 = 0.01%。Illumina 新平台大多 Q30+。
Phred formula: Q = -10·log10(P_error). Q20 = 1%, Q30 = 0.1%, Q40 = 0.01%. Modern Illumina is mostly Q30+.

FastQC 與 MultiQC

FastQC 是業界標準的單樣本品質報告產生器；MultiQC 則把多個 FastQC（或 STAR、Salmon、…）的 log 整合成一份可瀏覽的 HTML。

FastQC is the industry-standard per-sample QC report. MultiQC aggregates FastQC (and STAR, Salmon, …) logs into one browsable HTML.

⚠️
避免過度 trimming：Q20、min length 25–36 通常足夠。過度嚴格會丟失大量訊號，且現代比對工具（如 STAR、HISAT2）對輕微低品質鹼基有容忍度。
Avoid over-trimming: Q20 and min length 25–36 are usually enough. Overly strict thresholds discard real signal; modern aligners (STAR, HISAT2) tolerate slightly low-quality bases.
程式碼

實作範例

# 推薦：fastp 一站式 QC + trimming + 報告
fastp \
  -i sample_R1.fastq.gz -I sample_R2.fastq.gz \
  -o trim_R1.fastq.gz   -O trim_R2.fastq.gz \
  --detect_adapter_for_pe \
  --qualified_quality_phred 20 \
  --length_required 36 \
  --thread 8 \
  --html sample.fastp.html --json sample.fastp.json
# trim_galore 包裝 cutadapt，自動偵測接頭
trim_galore \
  --paired --illumina \
  --quality 20 --length 36 \
  --cores 4 --fastqc \
  -o trimmed/ \
  sample_R1.fq.gz sample_R2.fq.gz
# 跑完 FastQC 後一鍵彙整
fastqc -o fastqc_out/ *.fastq.gz
multiqc fastqc_out/ trimmed/ -o multiqc_report/
# 開啟 multiqc_report/multiqc_report.html

工具選擇

🌳 哪個 trimming 工具？

速度優先：
fastp — 多執行緒 + 一次完成 QC、trim、報告。
簡單易用：
trim_galore — 自動偵測接頭，預設參數即可上手。
精細調控：
cutadapt / Trimmomatic — 需要客製接頭序列或滑動視窗時的選擇。
UMI 處理：
umi_tools extract 先抽 UMI，再 trim。
Speed-first:
fastp — multi-threaded, one-shot QC + trim + report.
Easy default:
trim_galore — auto adapter detection; defaults work out of box.
Fine control:
cutadapt / Trimmomatic — for custom adapters or sliding-window logic.
UMI handling:
umi_tools extract first, then trim.
