對齊到參考基因組

把 reads 對應到基因組或轉錄組的位置——這是 RNA-seq 流程中計算成本最高、決策最深遠的一步。

Map reads to genome / transcriptome positions — the most compute-intensive, most consequential step of RNA-seq.

🟢 兩大派別 🟡 解讀輸出 🔴 演算法 + bias
核心概念

一個 splicing 帶來的麻煩

DNA-seq 比對相對簡單：read 應該對應到基因組上「一段連續區域」。但 RNA-seq 不一樣——mRNA 是剪接Splicing：pre-mRNA 中 intron 被切掉、exon 接在一起的過程。一條成熟 mRNA 通常由 5–15 個 exon 組成。後的產物，所以一條 read 可能橫跨兩個 exon、中間穿越一段 intron（即「junction read」）。傳統 DNA-seq 比對器（BWA、Bowtie2）會把這種 read 判定為「無法比對」或「軟剪」。

因此 RNA-seq 需要 splice-aware aligner——它知道參考基因組的 GT-AG splice site，可以把一條 read 拆成兩段、分別貼到兩個 exon。STAR 與 HISAT2 是現代兩大代表。

而 2014 年 Salmon、kallisto 引入另一條路：不算 read 的「位置」，只算 read 「屬於哪一個轉錄本」。這叫 pseudo-alignmentPseudo-alignment / Quasi-mapping：把 reads 拆成 k-mer，用 hash 表查出每個 k-mer 屬於哪些 transcript，最後用 EM 估計每個 transcript 的表達量。不輸出基因組座標。，速度比 STAR 快 10–100 倍、記憶體小到能在筆電上跑。

DNA-seq alignment is relatively simple: a read maps to one contiguous region of the genome. RNA-seq is different — mRNA is the splicedSplicing: introns are removed and exons joined together from pre-mRNA. A mature mRNA typically consists of 5–15 exons. product, so a single read may span two exons across an intron (a "junction read"). Traditional DNA aligners (BWA, Bowtie2) flag these as unmappable or soft-clip them.

Hence RNA-seq needs splice-aware aligners that know GT-AG splice donor/acceptor sites and can split a read across two exons. STAR and HISAT2 are the modern leaders.

In 2014 Salmon and kallisto introduced another path: skip exact positions, just record which transcripts a read could come from. This is pseudo-alignmentPseudo-alignment / quasi-mapping: break reads into k-mers, hash-lookup which transcripts each k-mer maps to, then use EM to estimate transcript abundances. No genome coordinates output. — 10–100× faster than STAR, with memory small enough to run on a laptop.

比喻： Splice-aware alignment 就像把每位旅客精確地分配到飛機座位（要知道 row 5、seat A）；pseudo-alignment 則是「我只想知道你搭哪個航班，座位不重要」——後者快得多，但若你想做 splicing 分析（座位視圖），就得用前者。 Splice-aware alignment is like assigning each passenger to a precise seat (row 5, seat A); pseudo-alignment is "I only care which flight you took, seat doesn't matter" — far faster, but you need the former if you want splicing analysis (seat-level view).

演算法直覺

背後在做什麼？

STAR（Spliced Transcripts Alignment to a Reference）：拿到一條 read，先在基因組找「最長能完美匹配的種子」，然後從種子向兩邊延伸。如果延伸時碰到 intron，就跳過 intron 繼續貼。最後輸出每條 read 的座標到 BAM。

Salmon：把 read 切成 31 bp 的「k-mer」，用 hash 表快速查每個 k-mer 屬於哪些 transcript（不需要真的比對）。當很多 k-mer 都指向同一個 transcript，就把這條 read 算給那個 transcript。多重比對的 reads 用 EM 演算法分攤。

STAR (Spliced Transcripts Alignment to a Reference): given a read, finds the "longest perfectly matching seed" in the genome, then extends in both directions. If extension hits an intron, jumps over and continues. Outputs each read's coordinates to BAM.

Salmon: chops the read into 31 bp "k-mers", hash-lookups which transcripts each k-mer matches (no actual alignment). When many k-mers point to the same transcript, the read is assigned there. Multi-mapping reads are split via EM.

STAR 的 Maximum Mappable Prefix (MMP)：從 read 5' 端開始，用 Suffix ArraySuffix Array：將參考基因組的所有 suffix 字典序排列。可在 O(L log G) 時間查詢任意字串。STAR 索引就是 GRCh38 的完整 Suffix Array，約 30 GB。 一次比對最長能匹配的前綴。匹配失敗就跳到下一個位置重新開始 MMP。剩下的零碎片段最後做 stitching。整個邏輯是 seed-and-extend，但 seed 不是固定長度而是「能匹配多長就多長」。

Salmon 的 quasi-mapping + EM：k-mer hash 找到候選 transcript（每條 read 有 1–N 個候選），再用 SMEM (super-maximal exact matches) 驗證真正的 mappability。最終透過 EM 解 「每個 read 應分給哪個 transcript」的 fractional assignment——這就是估計 TPM 的數學基礎。

STAR's Maximum Mappable Prefix (MMP): from the read's 5' end, find the longest matchable prefix using a Suffix ArraySuffix Array: all suffixes of the reference sorted lexicographically. Enables O(L log G) lookup. STAR's index is the full GRCh38 Suffix Array, ~30 GB.. On mismatch, jump to next position and restart MMP. Leftover fragments are stitched together. This is seed-and-extend but seeds have variable length.

Salmon's quasi-mapping + EM: k-mer hash finds candidate transcripts (1–N candidates per read), validated by SMEM (super-maximal exact matches). Final fractional assignment via EM — this is the math foundation of TPM estimation.

STAR 細節：用 uncompressed Suffix Array + auxiliary lookup tables (SA → genome chromosome/position mapping)，這就是 30GB 記憶體的來源。Splice junction discovery 分兩階段：① 從 sjdbGTFfile 載入已知 junction；② 從 read 中發現新的 junction（GT-AG canonical donor/acceptor）。--twopassMode Basic 會先掃一次抓 novel junctions，再用這些 junction 第二次比對，提升新 isoform 偵測準確度。

Salmon 細節：使用 selective alignment（v1.0+）替換早期的 pure k-mer，會對 k-mer 候選做 chain & extend 驗證，準確度接近 STAR。--gcBias 用 fragment 起點與終點的 GC 含量建立 logistic regression，把 PCR 偏差校正回估計值；--seqBias 校正 random hexamer priming bias。Posterior 不確定度透過 bootstrap (Gibbs sampling) 估計，可用於 sleuth (Pimentel et al., 2017) 的 isoform-level DE。

HISAT2 的 Graph-FM Index：以 graph 表示參考（含 SNPs / known splice sites），FM-index 可在 O(L) 完成查詢，記憶體 ~8 GB（遠低於 STAR），但 splice junction discovery 對 novel 弱於 STAR。

STAR internals: uncompressed Suffix Array + auxiliary lookup tables (SA → chromosome/position) — that's the 30 GB. Splice junction discovery is two-stage: (1) load known junctions from sjdbGTFfile; (2) discover novel junctions (GT-AG canonical donor/acceptor) from reads. --twopassMode Basic first-passes to collect novel junctions, then realigns using them, boosting novel isoform detection.

Salmon internals: uses selective alignment (v1.0+) instead of pure k-mer, performing chain & extend validation on k-mer candidates — accuracy approaching STAR. --gcBias builds a logistic regression on fragment-end GC content to correct PCR bias; --seqBias corrects random hexamer priming bias. Posterior uncertainty is estimated via bootstrap (Gibbs sampling), usable in sleuth (Pimentel et al., 2017) for isoform-level DE.

HISAT2's Graph-FM Index: represents the reference as a graph (SNPs + known splice sites). FM-index gives O(L) queries with ~8 GB memory (much less than STAR), but novel splice junction discovery is weaker.

參考檔案

選對 reference + GTF

「reference」這個詞容易讓人混淆，因為它有兩個檔案、且 RNA-seq 流程兩者都要：

"Reference" actually means two files in RNA-seq, and you need both:

檔案是什麼來源
Genome FASTA (.fa)基因組序列Ensembl primary_assembly / GENCODE
Annotation GTF (.gtf)基因註釋GENCODE（推薦）/ Ensembl
Transcript FASTA (for Salmon)轉錄本序列GENCODE pc_transcripts
GENCODE 比 Ensembl 好用： 兩者其實同步，但 GENCODE 提供 chrM、chr1...chrY 的命名（Ensembl 用 1, 2, ..., MT），與 IGV、UCSC、TCGA 命名一致，省去重命名步驟。 GENCODE and Ensembl share the same data, but GENCODE uses chrM, chr1...chrY (Ensembl uses 1, 2, ..., MT) — matching IGV, UCSC, TCGA conventions, saving rename headaches.
陷阱： 用了 patch-level 不同的 reference / GTF（如 GRCh38.p13 genome + p10 GTF），會導致少數基因座標對不上、counts 全為 0。genome 與 GTF 必須來自同一發行版。 Mixing patch levels (e.g., GRCh38.p13 genome + p10 GTF) shifts coordinates for some genes → counts = 0. Genome and GTF MUST come from the same release.
程式碼

完整實作範例

# 1. 建索引（一次性，~1 hr, 30GB RAM）
STAR --runMode genomeGenerate \
  --genomeDir star_idx/ \
  --genomeFastaFiles GRCh38.primary_assembly.genome.fa \
  --sjdbGTFfile gencode.v44.annotation.gtf \
  --sjdbOverhang 100 \
  --runThreadN 12

# 2. 兩階段比對 (推薦：可發現新 splice junctions)
STAR --runThreadN 12 \
  --genomeDir star_idx/ \
  --readFilesIn trim_R1.fq.gz trim_R2.fq.gz \
  --readFilesCommand zcat \
  --outSAMtype BAM SortedByCoordinate \
  --outSAMunmapped Within \
  --quantMode GeneCounts \
  --twopassMode Basic \
  --outFileNamePrefix sample1. \
  --outFilterMultimapNmax 20 \
  --alignSJoverhangMin 8 \
  --alignSJDBoverhangMin 1

# 重點輸出 / Key outputs:
#   sample1.Aligned.sortedByCoord.out.bam    ← downstream featureCounts
#   sample1.ReadsPerGene.out.tab             ← STAR 內建 gene count
#   sample1.SJ.out.tab                       ← splice junctions
#   sample1.Log.final.out                    ← QC metrics（給 MultiQC）
# 建索引（從 UCSC 預建可省時：grch38_genome.tar.gz）
hisat2-build -p 12 GRCh38.fa hisat_idx

# 比對 + sort BAM
hisat2 -p 12 -x hisat_idx \
  -1 trim_R1.fq.gz -2 trim_R2.fq.gz \
  --rna-strandness RF \
  --no-unal \
  --summary-file sample1.summary.txt \
  --new-summary \
  | samtools sort -@ 8 -o sample1.bam -

samtools index sample1.bam

# --rna-strandness RF = dUTP stranded (reverse strand for R1)
# 若 unstranded library：省略此旗標
# 1. 建轉錄本索引（含 decoy，~10 min, 4GB RAM）
# decoy = 整個 genome，幫忙吸收會錯誤對到 intronic / intergenic 的 reads
grep "^>" GRCh38.fa | cut -d " " -f 1 | sed 's/>//g' > decoys.txt
cat gencode.v44.transcripts.fa GRCh38.fa > gentrome.fa

salmon index -t gentrome.fa --decoys decoys.txt \
  -i salmon_idx -k 31 --threads 12

# 2. 量化（自動偵測 library type；GC + sequence bias correction）
salmon quant -i salmon_idx -l A \
  -1 trim_R1.fq.gz -2 trim_R2.fq.gz \
  --validateMappings \
  --gcBias --seqBias \
  --numGibbsSamples 20 \
  --threads 8 \
  -o sample1_salmon

# 重點輸出：
#   sample1_salmon/quant.sf       ← transcript-level TPM + NumReads
#   sample1_salmon/lib_format_counts.json
#   sample1_salmon/aux_info/bootstrap/   ← Gibbs samples
# 用 tximport 整合到 gene-level → DESeq2
# kallisto 流程更簡單（無 decoy 機制）
kallisto index -i kall_idx gencode.v44.transcripts.fa

kallisto quant -i kall_idx \
  -o sample1_kall \
  --rf-stranded \
  -t 8 -b 100 \
  trim_R1.fq.gz trim_R2.fq.gz

# -b 100 = 100 次 bootstrap，供 sleuth 估計 isoform-level uncertainty
# 1. Picard CollectRnaSeqMetrics (3'/5' bias、ribosomal %)
picard CollectRnaSeqMetrics \
  I=sample1.bam \
  O=sample1.rnaseq.metrics \
  REF_FLAT=refFlat.txt \
  STRAND=SECOND_READ_TRANSCRIPTION_STRAND \
  RIBOSOMAL_INTERVALS=rRNA.interval_list

# 2. RSeQC infer_experiment.py 驗證 strandedness
infer_experiment.py -r gencode.v44.bed -i sample1.bam

# 3. 對所有 BAM、log、metrics 跑 MultiQC
multiqc star_out/ picard_out/ rseqc_out/ -o multiqc/

# 4. samtools flagstat 看 PE/PE proper、duplicate、mapped
samtools flagstat sample1.bam > sample1.flagstat.txt

CASE STUDY

Salmon vs STAR：誰才是「正確答案」？

2017 年 Robert Patro 發表 Salmon (Nat. Methods)，引發了一場「pseudo-alignment 夠不夠準」的論戰。Soneson 等人 (F1000Research 2016) 比較了 Salmon、kallisto、STAR+featureCounts 在 SEQC dataset 與模擬數據上的表現：

  • Gene-level DE：三者結果高度一致（Spearman ρ > 0.95），DEG 重疊 > 90%。
  • Isoform-level：Salmon 與 kallisto 比 STAR+RSEM 略遜於精確 isoform 估計，但 EM 演算法穩定。
  • Bias correction：Salmon 的 --gcBias、--seqBias 在高 GC、低 GC 基因上顯著減少假陽性。
  • 節能：30M PE 樣本 Salmon 用 0.5 CPU-hr，STAR 用 5 CPU-hr——對大計畫（> 100 樣本）差距顯著。

結論：對 gene-level 表達 DE，Salmon 與 STAR 等價；速度與資源優勢使 Salmon 成為近年 best practice。但若你的研究關心 splicing、需要 BAM、要做變異偵測，STAR 仍不可取代。

The 2017 Salmon paper (Patro et al., Nat. Methods) sparked an "is pseudo-alignment accurate enough?" debate. Soneson et al. (F1000Research 2016) benchmarked Salmon, kallisto, STAR+featureCounts on SEQC and simulated data:

  • Gene-level DE: all three highly concordant (Spearman ρ > 0.95), DEG overlap > 90%.
  • Isoform-level: Salmon and kallisto slightly trailed STAR+RSEM on precise isoform estimation, but EM is stable.
  • Bias correction: Salmon's --gcBias + --seqBias markedly reduced false positives on high/low-GC genes.
  • Energy: 30M PE sample: Salmon 0.5 CPU-hr, STAR 5 CPU-hr — huge difference at scale (> 100 samples).

Conclusion: for gene-level DE, Salmon ≈ STAR; speed/resource benefits made Salmon current best practice. But splicing, BAM, and variant calling still require STAR.

陷阱

比對常見錯誤

① 用 BWA / Bowtie2 比對 RNA-seq
這些是 DNA aligner，無法跨 splice junction，會把 ~30% 的 junction reads 丟掉或軟剪。永遠用 splice-aware aligner（STAR / HISAT2）。
These are DNA aligners — they cannot span splice junctions and will drop or soft-clip ~30% of junction reads. Always use splice-aware aligners (STAR / HISAT2).
② Salmon 沒加 decoy
不加 decoy（即不把 genome 加進 transcriptome FASTA），會讓本該對到 intron / intergenic 的 reads 強制分配給某個 transcript，造成假陽性。永遠用 gentrome.fa + decoys.txt。
Without decoy (i.e., not including the genome alongside the transcriptome), reads that should map to introns/intergenic get forced into transcripts → false positives. Always use gentrome.fa + decoys.txt.
③ Strandedness 設錯
最常見：dUTP library 卻設 -s 0 或 -s 1。結果：counts 約剩一半，DEGs 全是假陽性。永遠先跑 infer_experiment.py 驗證。
Most common: dUTP library set to -s 0 or -s 1. Result: half the counts, mostly false DEGs. Always verify with infer_experiment.py first.
④ Genome / GTF patch 不一致
GRCh38.p13 genome + p10 GTF 會讓部分 gene 找不到對應 exon。下載時務必一同從同一個 GENCODE release 取。
GRCh38.p13 genome + p10 GTF means some genes can't find their exons. Always download genome + GTF from the same GENCODE release.
⑤ STAR --sjdbOverhang 設錯
--sjdbOverhang 應該設為 read length − 1。常見錯誤：用 PE 150 卻仍用 STAR 預設 100。會導致長 read 的 junction 偵測力下降 5–10%。
--sjdbOverhang should be read length − 1. Common mistake: using PE 150 with STAR's default 100 → 5–10% drop in junction detection for long reads.

加快與改善比對品質

① 索引建一次，全實驗室共用 STAR index 30 GB / Salmon 5 GB，建一次能用多年（只要 GENCODE 版本不變）。實驗室伺服器上 /data/reference/star_idx_v44/ 共享路徑。 STAR index 30 GB / Salmon 5 GB — build once, reuse for years (as long as GENCODE version stays put). Use a shared lab path like /data/reference/star_idx_v44/.
② STAR + Salmon 雙跑（推薦） STAR 拿 BAM 做 QC + IGV 視覺化 + variant calling；Salmon 從 BAM (--alignment-based) 拿 TPM + bias correction。最佳兼顧速度與分析靈活性。 STAR for BAM (QC + IGV + variants); Salmon in --alignment-based mode on the BAM for bias-corrected TPMs. Best of both worlds.
③ 多執行緒不是越多越好 STAR 在 16 threads 後速度幾乎飽和；Salmon 在 12 threads 後飽和。把 64-core server 開 64 個 process 比 1 個 process 用 64 threads 還快。 STAR saturates at ~16 threads; Salmon at ~12. A 64-core server is faster with 64 single-thread processes than 1 process with 64 threads.
④ 用 nf-core/rnaseq 或 snakemake 生產環境不要手寫 for-loop。nf-core/rnaseq 已內建 fastp + STAR + Salmon + featureCounts + MultiQC，並可在 SLURM/AWS 上自動排程。 For production, don't write for-loops. nf-core/rnaseq bundles fastp + STAR + Salmon + featureCounts + MultiQC and auto-schedules on SLURM/AWS.

常見問題

STAR 把整個 Suffix Array 載入記憶體 (~25 GB) + auxiliary lookup tables (~5 GB)。這是 O(1) 查詢的代價。HISAT2 用壓縮過的 Graph-FM Index 只要 8 GB，但查詢稍慢。

STAR loads the full Suffix Array (~25 GB) + auxiliary tables (~5 GB) into RAM — the price of O(1) lookups. HISAT2 uses a compressed Graph-FM Index (~8 GB) but slower lookup.

Salmon 用 --writeMappings 可以額外輸出 SAM，但這違背了 pseudo-alignment 的速度優勢。實務上要 IGV/變異偵測就用 STAR；Salmon 專注定量。

Salmon's --writeMappings can write SAM, but that defeats the pseudo-alignment speedup. For IGV / variants, use STAR; reserve Salmon for quantification.

不是強制，但建議做。第一階段只用 known junctions；第二階段加入發現的 novel junctions 重比一次。對 splicing 研究、新 isoform 發現有幫助；對純 gene-level DE 影響微小。

Not mandatory but recommended. Pass 1 uses known junctions only; pass 2 adds discovered novel junctions and re-aligns. Helps splicing / novel isoform discovery; minor impact on gene-level DE.

2023 後共識：Salmon。原因：① decoy-aware selective alignment 更準；② --gcBias + --seqBias 校正；③ posterior bootstrap 較完整。kallisto 仍很好用，但無 bias correction。

2023 consensus: Salmon. (1) decoy-aware selective alignment is more accurate; (2) --gcBias + --seqBias; (3) richer posterior bootstrap. kallisto is still good but lacks bias correction.

featureCounts 預設丟棄 (-M 0)；STAR ReadsPerGene 也丟棄。Salmon 用 EM 分數攤分（保留訊號）。建議：DE 分析丟棄即可；若關心 paralog/HLA，用 Salmon 的 EM。

featureCounts drops them by default (-M 0); STAR ReadsPerGene also drops. Salmon partitions them via EM (preserving signal). Recommendation: drop for DE; use Salmon's EM if studying paralogs/HLA.

常見三大原因：① rRNA 汙染（poly-A 失敗）→ 用 FastQ Screen 查；② 跨物種汙染（如人類樣本汙染了小鼠）；③ 用錯 reference（如 GRCh37 vs GRCh38）。

Three usual suspects: (1) rRNA contamination (failed poly-A) → check with FastQ Screen; (2) cross-species contamination (e.g., mouse in human sample); (3) wrong reference (GRCh37 vs GRCh38).
