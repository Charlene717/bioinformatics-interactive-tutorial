比對到參考基因組

把 reads 對應到基因組或轉錄組的位置——是 RNA-seq 流程中計算成本最高、結果影響最大的一步。

Map reads to genome / transcriptome positions — the most compute-intensive, results-defining step of RNA-seq.

核心概念

兩大派別

現代 RNA-seq 量化大致分為兩類：

Modern RNA-seq quantification splits into two families:

🎯

拼接式比對

把每條 read 確切定位到基因組 BAM。代表：STAR、HISAT2。輸出 BAM 可用於變異偵測、視覺化 (IGV)、SJ 分析等下游。

Maps every read to genome coordinates as BAM. Tools: STAR, HISAT2. BAM enables variant calling, IGV visualization, splice junction analysis.

⚡

偽比對

只記 read 屬於哪個轉錄本，不算精確位置。代表：Salmon、kallisto。速度快 10–100×、記憶體小，直接輸出表達量。

Only records which transcripts each read could come from, not exact positions. Tools: Salmon, kallisto. 10–100× faster, low memory, direct quantification.

FASTQ→ STAR / HISAT2→ BAM→ featureCounts→ counts
FASTQ→ Salmon / kallisto→ quant.sf / abundance→ tximport→ DESeq2

業界基準：RNA-seq 樣本通常需要至少 20 M usable reads（去除 rRNA、重複後）才能可靠偵測中低表達基因；ENCODE 建議 30 M paired-end。

程式碼

實作範例

# 1. 建索引（一次性）
STAR --runMode genomeGenerate \
  --genomeDir star_idx/ \
  --genomeFastaFiles GRCh38.fa \
  --sjdbGTFfile gencode.v44.gtf \
  --sjdbOverhang 100 --runThreadN 12

# 2. 比對
STAR --runThreadN 12 \
  --genomeDir star_idx/ \
  --readFilesIn R1.fq.gz R2.fq.gz \
  --readFilesCommand zcat \
  --outSAMtype BAM SortedByCoordinate \
  --quantMode GeneCounts \
  --outFileNamePrefix sample1.
# 建索引（從 UCSC 預建可省時）
hisat2-build GRCh38.fa hisat_idx

# 比對 + sort
hisat2 -p 12 -x hisat_idx \
  -1 R1.fq.gz -2 R2.fq.gz \
  --rna-strandness RF \
  --summary-file sample1.summary.txt \
  | samtools sort -@ 8 -o sample1.bam -
# 1. 建轉錄本索引（含 decoy）
salmon index -t gentrome.fa --decoys decoys.txt \
  -i salmon_idx -k 31 --threads 12

# 2. 量化（自動偵測 library type）
salmon quant -i salmon_idx -l A \
  -1 R1.fq.gz -2 R2.fq.gz \
  --validateMappings \
  --gcBias --seqBias \
  -p 12 -o sample1_salmon
進階

比對品質指標

Uniquely mapped %

良好樣本應 > 75%。若 < 60%，常見原因：rRNA 汙染、宿主／非預期生物汙染、index 錯誤。

Healthy samples >75%. Below 60% often means rRNA contamination, foreign-organism content, or wrong reference.

% mapped to too many loci

> 5% 表示 rRNA 或重複序列汙染。

>5% suggests rRNA or repetitive content.

3' / 5' 偏差

用 Picard CollectRnaSeqMetrics。3' bias 高 → RNA 降解嚴重；建議 RIN ≥ 7。

Use Picard CollectRnaSeqMetrics. High 3' bias → severe RNA degradation; aim for RIN ≥ 7.
