開場故事

為什麼不能用 t-test？

1990 年代的 microarray 時代，研究者習慣對每個基因做 t-test。結果呢？同一個資料用不同軟體跑出來的「DEG」重疊度常低於 30%。原因：t-test 假設常態分布 + 等變異，但基因表達是離散整數、且 variance 隨 mean 變化（高表達基因變異大、低表達基因變異小）。

RNA-seq counts 是 離散整數Discrete integers. counts 不會出現 3.7 reads；最低就是 0。，且呈現 過度離散Over-dispersion: variance > mean，違反 Poisson 假設（var=mean）。負二項分佈專門處理此特性。（variance > mean）。t-test 在這上面會嚴重低估 false positives。三大主流 DE 工具（DESeq2、edgeR、limma-voom）用不同方式解決這問題，但都基於統計學上更嚴謹的負二項或加權線性模型。

2014 年 DESeq2 論文發表至今，引用超過 5 萬次——它徹底改變了 RNA-seq 分析的可重現性。每一個 RNA-seq 工作者都應該理解：「為什麼選這個工具」比「會用這個工具」更重要。

In the 1990s microarray era, researchers ran t-tests on each gene. The result? The same dataset analysed by different tools agreed on < 30% of "DEGs." The reason: t-test assumes normality + equal variance, but gene expression is integer-valued and has variance that scales with mean (high expressors are more variable than low ones).

RNA-seq counts are discrete integersYou never see 3.7 reads; the minimum is 0. with over-dispersionvariance > mean — violates the Poisson assumption (var = mean). Negative binomial handles this.. T-test severely under-estimates false positives. The three mainstream DE tools (DESeq2, edgeR, limma-voom) each solve this differently but all use statistically rigorous negative binomial or weighted linear models.

Since DESeq2's 2014 publication it has > 50,000 citations — it fundamentally changed RNA-seq reproducibility. Every RNA-seq analyst should understand: "why this tool" matters more than "how to use this tool."

類比：t-test 像「直接量花瓣長度比較兩種花」；DE 工具像「同時考慮花的整體大小、種子大小、平均壽命等變異模式」。後者更貼近生物學真實。
Analogy: t-test is like comparing flowers by petal length alone; DE tools consider overall plant size, seed size, lifespan variability — closer to biological reality.

一、三大模型的核心邏輯

Tool分佈假設變異處理特色最佳情境
DESeq2Negative BinomialEB 收縮預設工具小到中樣本

ygi ~ NegBin(μgi, αg)
log(μgi) = β0 + β1·conditioni + log(si) DESeq2 核心模型

意思：基因 g 在樣本 i 的 read count ygi 服從負二項分佈，平均值 μgi 來自條件效應 + library size factor si，αg 是過度離散參數（每個基因獨立估計、用 EB shrink）。DE 檢定 = β1 ≠ 0 的 Wald 檢定。

Meaning: gene g's count ygi in sample i follows a negative binomial with mean μgi from condition effect + library size factor si, where αg is the gene-specific dispersion (EB-shrunk). The DE test is a Wald test of β1 ≠ 0.

統計輸出

二、解讀 DE 表格

log2FoldChange (LFC)

對數轉換的倍數變化。LFC = 1 = 2 倍上升；LFC = -1 = 減半。shrunken LFC（apeglm / ashr）會把低 counts 基因高估的 LFC 拉回真實值。

Log fold change. LFC = 1 = 2× up; LFC = -1 = half. Shrunken LFC (apeglm / ashr) tames inflated LFCs from low-count genes.

p-value

H₀「無差異」下觀察到此差異的機率，未校正多重比較。20,000 基因下，p < 0.05 也有 1000 個假陽性。

Probability under H₀ "no change," not multiple-testing corrected. With 20,000 genes, p < 0.05 still yields ~1000 false positives.

padj (FDR)

BH 校正後的調整 p 值。真正的「顯著」門檻通常用 padj < 0.05 或 0.1。

BH-adjusted p-value. Use padj < 0.05 or 0.1 for "significance".

baseMean / AveExpr

該基因跨所有樣本的平均表達。極低 baseMean (< 10) 即使 LFC 大也要謹慎。

Mean expression across samples. Very low baseMean (< 10) warrants caution even with large LFC.

padj = BH(p)
Benjamini-Hochberg: padj(i) = min(p(i) · m / i, padj(i+1)) FDR 校正

案例研究

四、真實案例：airway 資料集

Himes et al. 2014 · PLOS ONE · GSE52778

📂 dexamethasone vs control

真實資料 Himes 等人收集 4 位捐贈者的人類氣管平滑肌細胞，分別處理 dexamethasone（一種類固醇）24 小時與 control。8 個樣本（4 paired pairs）。原始問題：糖皮質激素如何抑制氣道發炎？

結果：DESeq2 找到 ~2,500 個 DEGs（padj < 0.1）。Top up-regulated 基因包含已知的糖皮質激素標靶 DUSP1、KLF15；top down-regulated 包含 inflammatory cytokine IL6、CXCL5。這完全符合預期的「類固醇抗發炎」生物學。

教學價值：(1) 公開且乾淨；(2) 樣本配對（subject）需用 design = ~ subject + condition；(3) Bioconductor 套件 airway 直接可用，零環境設定。每個 DESeq2 vignette 都用它示範。

Real Data Himes et al. collected human airway smooth muscle cells from 4 donors, treated with dexamethasone (a corticosteroid) 24 h vs control. 8 samples (4 paired). Original question: how do glucocorticoids suppress airway inflammation?

Result: DESeq2 found ~2,500 DEGs (padj < 0.1). Top up-regulated genes include known glucocorticoid targets DUSP1, KLF15; top down-regulated include inflammatory cytokines IL6, CXCL5. Perfectly consistent with steroid anti-inflammatory biology.

Teaching value: (1) public + clean; (2) paired samples (subject) need design = ~ subject + condition; (3) Bioconductor package airway ships ready-to-use, zero setup. Every DESeq2 vignette uses it.

程式碼

五、完整 DE 流程

library(DESeq2)
# 1. 建 dds 物件
dds <- DESeqDataSetFromMatrix(countData = cts,
                              colData   = coldata,
                              design    = ~ batch + condition)
# 2. 過濾低 counts（可選但建議）
dds <- dds[rowSums(counts(dds)) >= 10, ]
# 3. 跑 DE（size factor、dispersion、Wald test 一次到位）
dds <- DESeq(dds)
res <- results(dds, contrast = c("condition", "treated", "control"),
               alpha = 0.05)
# 4. LFC shrinkage（推薦：給排名用）
res_s <- lfcShrink(dds, coef = "condition_treated_vs_control", type = "apeglm")
summary(res_s)
library(edgeR)
y <- DGEList(counts = cts, group = grp)
keep <- filterByExpr(y, group = grp)
y <- y[keep, , keep.lib.sizes = FALSE]
y <- calcNormFactors(y, method = "TMM")
design <- model.matrix(~ batch + condition)
y <- estimateDisp(y, design)
fit <- glmQLFit(y, design)
qlf <- glmQLFTest(fit, coef = "conditiontreated")
tt  <- topTags(qlf, n = Inf)$table
library(limma); library(edgeR)
dge <- DGEList(cts)
keep <- filterByExpr(dge, group = grp)
dge <- dge[keep, , keep.lib.sizes = FALSE]
dge <- calcNormFactors(dge)
design <- model.matrix(~ batch + condition)
v   <- voom(dge, design, plot = TRUE)
fit <- lmFit(v, design)
fit <- eBayes(fit)
tt  <- topTable(fit, coef = "conditiontreated", n = Inf)
進階

六、複雜實驗設計

批次校正

把 batch 放進 design（如 ~ batch + condition），DESeq2/limma 會在模型內校正。若僅作視覺化，可用 limma::removeBatchEffect()。

Put batch in design (e.g., ~ batch + condition) — DESeq2/limma adjust internally. For viz only, use limma::removeBatchEffect().

多組比較與交互作用

用 contrast = c(...) 對任意組合取差異。交互作用以 ~ genotype * treatment 編碼。

Use contrast = c(...) for any pairwise comparison. Interactions via ~ genotype * treatment.

配對 / 重複測量

配對樣本：~ subject + condition。limma 可用 duplicateCorrelation() 處理重複測量。

Paired: ~ subject + condition. In limma, duplicateCorrelation() handles repeated measures.

隱藏干擾

用 sva 或 RUVSeq 偵測未記錄的批次源。把推得的 surrogate variables 加進 design。

Use sva or RUVSeq to detect unrecorded batch sources; add the SVs to the design.

討論

八、思考題

💭 五道值得思考的問題

Q1. 你的 cohort 有 4 個 case、4 個 control，PCA 顯示 PC1 沿性別分群、case/ctrl 沿 PC3。原始 design 是 ~ condition，怎麼修改？討論：要加 sex？要把 sex 與 condition 解耦的代價？

Q2. 為什麼 DESeq2 對 padj=NA 的基因「不算進 BH」？這在低表達基因上特別常見。提示：independent filtering。

Q3. 兩個樣本 cohort，case 與 ctrl 各 3 隻小鼠。如果一隻 case 死亡只剩 n=2 vs 3，你會：(a) 用剩餘樣本做 DE、(b) 把它變成 n=3 vs 3 的 paired design 重新分析、(c) 找新合作者重做？討論各方案的科學成本。

Q4. Bulk RNA-seq cohort 與 scRNA-seq pseudo-bulk 在同一條件比較下，DEG 一致度通常 30-50%。為什麼？想想 cell-type composition、dropout、normalization 的差異。

Q5. DESeq2 的 LFC shrinkage 對「真正高 fold-change 但 baseMean 低的基因」會壓抑訊號。在罕見 marker gene 研究時，要不要用 shrunken LFC？討論 ranking vs reporting 的差別。

Q1. Cohort: 4 case + 4 control. PCA shows PC1 follows sex; case/ctrl separate along PC3. Original design ~ condition. How to fix? Discuss: add sex? Cost of decoupling sex from condition?

Q2. Why does DESeq2 set padj=NA for some low-count genes and exclude them from BH? Hint: independent filtering.

Q3. Cohort: 3 mice per group; one case mouse dies leaving n=2 vs 3. Options: (a) DE on remaining; (b) redesign as paired n=3 vs 3; (c) find collaborator to redo. Discuss costs.

Q4. Bulk RNA-seq vs scRNA pseudo-bulk for the same condition typically agree on only 30–50% of DEGs. Why? Consider cell-type composition, dropout, normalization differences.

Q5. DESeq2's LFC shrinkage suppresses "high fold-change but low baseMean" genes. For rare marker discovery, use shrunken LFC? Discuss ranking-vs-reporting trade-off.

FAQ

九、常見問答

技術上可行，但結果不可靠。RNA-seq counts 違反常態與等變異假設。t-test 對基因數 > 5,000 的 DE 場景產生顯著膨脹的 false positives。除非用無母數版本（如 Mann-Whitney）+ 大樣本 (n > 30)，否則別用。

Technically possible, but unreliable. RNA-seq counts violate normality and equal-variance assumptions. T-test inflates false positives at genome scale. Unless you use non-parametric (e.g., Mann-Whitney) + large n (> 30), avoid it.

沒有絕對贏家。Schurch et al. (2016) 對 24 個工具的 benchmark：DESeq2 與 edgeR 表現幾乎相同。差異：

  • DESeq2 預設 LFC shrinkage → 排名更穩定
  • edgeR 對複雜 design (含 GLM contrasts) 更靈活
  • limma-voom 在大 cohort 速度勝出

建議：選一個你熟的，並理解其數學基礎。

No clear winner. Schurch et al. (2016) benchmarked 24 tools: DESeq2 and edgeR perform similarly. Differences:

  • DESeq2 has default LFC shrinkage → more stable ranking
  • edgeR is more flexible for complex GLM contrasts
  • limma-voom wins on speed for large cohorts

Recommendation: pick one you know, understand its math.

取決於生物學情境：

  • 強處理（如 dexamethasone、IFN 刺激）：1,000-5,000 DEGs
  • 溫和處理（mild stress、低劑量藥物）：100-1,000
  • 細微差異（年齡、輕度疾病）：< 500 或甚至 < 100
  • 0 DEGs：可能真實（無生物學差異）或統計力不足；用 power analysis 區分
  • > 10,000 DEGs：高度可疑——可能是 batch effect、normalization 失敗、或 PCA outlier 未移除

Depends on biology:

  • Strong perturbation (dexamethasone, IFN): 1,000–5,000
  • Mild perturbation: 100–1,000
  • Subtle (aging, mild disease): < 500 or even < 100
  • 0 DEGs: could be real (no biology) or under-powered; distinguish with power analysis
  • > 10,000 DEGs: highly suspicious — batch effect, normalization failure, or unremoved PCA outliers

視情境。在大 cohort (n > 30) 中，padj < 0.05 + LFC 0.3（~1.23×）統計上可信。但生物學上 1.23 倍變化是否「有意義」需個別評估。實驗驗證 (qPCR/Western) 通常需要 |LFC| > 1（2 倍）才能可靠複製。

It depends. In large cohorts (n > 30), padj < 0.05 + LFC 0.3 (~1.23×) is statistically credible. Whether 1.23× is biologically meaningful must be judged case-by-case. Wet-lab validation (qPCR/Western) typically needs |LFC| > 1 (2×) for reliable replication.

常見原因：(1) DESeq2 版本（每個 minor version 算法微調，記得鎖版本）；(2) 輸入順序（不同 sample 順序可能改 size factor 細節）；(3) filter 是否相同。為可重現性：用 sessionInfo() 記錄版本、固定 sample 排序、撰寫 Snakemake/nf-core pipeline。

Common causes: (1) DESeq2 version (minor updates tweak algorithms — lock the version); (2) input sample order (can subtly change size factors); (3) different filters. For reproducibility: record sessionInfo(), fix sample order, use Snakemake/nf-core pipelines.
