差異表達分析

Use statistical models to find genuinely changing genes between groups — the core purpose of RNA-seq.

核心概念

為什麼不用 t-test？

RNA-seq counts 是離散整數，且呈現 過度離散（variance > mean），t-test 假設常態分布 + 等變異，會嚴重低估 false positives。三大主流模型用不同方式解決：

RNA-seq counts are discrete integers and show over-dispersion (variance > mean). A t-test assumes normality and equal variance, which underestimates false positives. The three mainstream models handle this differently:

統計輸出

解讀 DE 表格

log2FoldChange (LFC)

對數轉換的倍數變化。LFC = 1 代表 2 倍上升，LFC = -1 代表減半。LFC shrinkage 會把低 counts 基因的高估倍數拉回真實值。

Log-transformed fold change. LFC = 1 means 2× up, LFC = -1 means half. LFC shrinkage tames inflated estimates from low-count genes.

p-value

在 H₀「無差異」下觀察到此差異的機率，未校正多重檢定。20,000 基因下，p < 0.05 也有 1000 個假陽性。

Probability under H₀ "no change". Not multiple-testing corrected. With 20,000 genes, p < 0.05 still yields ~1000 false positives.

padj (FDR)

經 BH 校正的調整 p 值。真正的「顯著」門檻通常用 padj < 0.05 或 0.1。

BH-adjusted p-value. Use padj < 0.05 or 0.1 for "significance".

baseMean / AveExpr

該基因跨所有樣本的平均表達。極低 baseMean 的基因即使 LFC 很大也要謹慎。

Mean expression across samples. Be cautious about genes with very low baseMean even if LFC is large.

程式碼

完整 DE 流程

library(DESeq2)
# 1. 建 dds 物件
dds <- DESeqDataSetFromMatrix(countData = cts,
                              colData   = coldata,
                              design    = ~ batch + condition)
# 2. 過濾低 counts（可選但建議）
dds <- dds[rowSums(counts(dds)) >= 10, ]
# 3. 跑 DE（size factor、dispersion、Wald test 一次到位）
dds <- DESeq(dds)
res <- results(dds, contrast = c("condition", "treated", "control"),
               alpha = 0.05)
# 4. LFC shrinkage（推薦：給排名用）
res_s <- lfcShrink(dds, coef = "condition_treated_vs_control", type = "apeglm")
summary(res_s)
library(edgeR)
y <- DGEList(counts = cts, group = grp)
keep <- filterByExpr(y, group = grp)
y <- y[keep, , keep.lib.sizes = FALSE]
y <- calcNormFactors(y, method = "TMM")
design <- model.matrix(~ batch + condition)
y <- estimateDisp(y, design)
fit <- glmQLFit(y, design)
qlf <- glmQLFTest(fit, coef = "conditiontreated")
tt  <- topTags(qlf, n = Inf)$table
library(limma); library(edgeR)
dge <- DGEList(cts)
keep <- filterByExpr(dge, group = grp)
dge <- dge[keep, , keep.lib.sizes = FALSE]
dge <- calcNormFactors(dge)
design <- model.matrix(~ batch + condition)
v   <- voom(dge, design, plot = TRUE)
fit <- lmFit(v, design)
fit <- eBayes(fit)
tt  <- topTable(fit, coef = "conditiontreated", n = Inf)
進階

複雜實驗設計

批次校正

把 batch 放進 design（如 ~ batch + condition），DESeq2/limma 會在模型內校正。若僅作視覺化，可用 limma::removeBatchEffect()。

Put batch in design (e.g., ~ batch + condition) — DESeq2/limma adjust for it. For viz only, use limma::removeBatchEffect().

多組比較與交互作用

用 contrast = c(...) 對任意組合取差異。交互作用以 ~ genotype * treatment 編碼。

Use contrast = c(...) for any pairwise comparison. Interactions via ~ genotype * treatment.

配對 / 重複測量

配對樣本：~ subject + condition。limma 可用 duplicateCorrelation() 處理。

Paired samples: ~ subject + condition. In limma, use duplicateCorrelation().

隱藏干擾

用 sva 或 RUVSeq 偵測未記錄的批次源。把推得的 surrogate variables 加進 design。

Use sva or RUVSeq to find unrecorded batch sources; add the SVs into the design.

📝 自我檢測

RNA-seq counts 的「過度離散」是指？

"Over-dispersion" in RNA-seq counts means?

A. 樣本之間的 library size 差異A. Library-size differences between samples
B. variance 遠大於 mean，違反 Poisson 假設B. Variance ≫ mean, violating Poisson assumption
C. 高表達基因比低表達基因多C. More highly than lowly expressed genes
D. 偵測到太多 DEGD. Too many DEGs detected

何時最該使用 DESeq2 的 lfcShrink()？

When is DESeq2's lfcShrink() most useful?

A. 取代 p-value 校正A. To replace p-value adjustment
B. 增加 DEG 數量B. To increase DEG count
C. 在依 LFC 排名挑 top 基因時，減低低 counts 基因的假象高 LFCC. When ranking by LFC — tame inflated LFCs from low-count genes
D. 改善樣本對齊率D. Improves sample alignment rate
