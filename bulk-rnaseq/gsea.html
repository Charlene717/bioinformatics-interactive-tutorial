功能富集 / GSEA

從「一堆 DEG」躍升到「哪些生物路徑被影響」——讓結果有可解讀的生物學意義。

Move from "a list of DEGs" to "which biological pathways are affected" — the bridge to biological meaning.

核心概念

ORA vs GSEA

🎯

Over-Representation Analysis (ORA)

先用 padj/LFC 取出 DEG 列表，再問：「這個 DEG list 是否富集某個基因集（如 GO、KEGG）？」用 hypergeometric (Fisher's exact) 檢定。

Take a DEG list using padj/LFC, ask: "Is this DEG list enriched for a gene set (GO, KEGG)?" Uses hypergeometric (Fisher's exact) test.

📈

Gene Set Enrichment Analysis (GSEA)

不切門檻，把所有基因依 stat (LFC × sign(LFC)) 排序，看某 gene set 是否「聚」在排名兩端。用 Kolmogorov–Smirnov-style enrichment score + 置換檢定。

No cutoff — rank all genes by a stat (LFC × sign), test whether a gene set clusters at the top/bottom of the ranked list with a KS-style score + permutation test.

💡
選擇原則：有明確 DEG list（如 padj < 0.05、|LFC| > 1）→ ORA；想知道整體趨勢（即使少有顯著 DEG）→ GSEA。實務上兩個都做。
Rule of thumb: Have a clear DEG list (padj < 0.05, |LFC| > 1)? → ORA. Want overall trends even without strong DEGs? → GSEA. In practice, do both.

實作範例

library(clusterProfiler); library(org.Hs.eg.db)
# --- ORA (GO BP) ---
deg <- res[res$padj < 0.05 & abs(res$log2FoldChange) > 1, ]
ora <- enrichGO(gene = rownames(deg),
                universe = rownames(res),
                OrgDb    = org.Hs.eg.db,
                keyType  = "SYMBOL",
                ont      = "BP",
                qvalueCutoff = 0.05)
dotplot(ora, showCategory = 15)

# --- GSEA (KEGG, ranked list) ---
ranks <- res$log2FoldChange
names(ranks) <- res$ENTREZ
ranks <- sort(ranks, decreasing = TRUE)
gsea <- gseKEGG(geneList = ranks, organism = "hsa",
                 nPerm = 10000, minGSSize = 15)
gseaplot2(gsea, geneSetID = 1)
library(fgsea); library(msigdbr)
# MSigDB Hallmark gene sets
h <- msigdbr(species = "Homo sapiens", category = "H")
sets <- split(h$gene_symbol, h$gs_name)
ranks <- sort(setNames(res$stat, rownames(res)), decreasing = TRUE)
fg <- fgsea(pathways = sets, stats = ranks,
            minSize = 15, maxSize = 500)
head(fg[order(padj), ], 10)

解讀

GSEA 結果指標

ES / NES

Enrichment Score 與其經 size-correction 後的 NES。NES > 0 = 富集於上調端、< 0 = 下調端。|NES| 越大越強。

Enrichment Score and size-corrected NES. NES > 0 = enriched in upregulated end; < 0 = downregulated. Larger |NES| = stronger.

p / padj (FDR-q)

置換檢定 p 值與 BH 調整後的 q。常用門檻 padj < 0.25（GSEA 原作建議；保守可用 0.05）。

Permutation p and BH-adjusted q. Common cutoff: padj < 0.25 (GSEA paper recommendation; stricter use 0.05).

Leading edge

在 ES 峰值之前的「驅動基因」子集——是真正撐起該路徑富集的 gene list。

The "leading-edge" subset — the genes before the ES peak that actually drive the enrichment.

陷阱

常見錯誤

🚫
ORA 用錯 background：universe 必須是「實驗中可被偵測的所有基因」，而非整個基因組或全部 KEGG 基因。用錯 background 會嚴重虛高 enrichment。
Wrong ORA background: universe must be "all genes detected in the experiment", not the whole genome or all KEGG genes. Wrong background inflates enrichment.
🚫
GSEA 用未排序、或包含 NA 的 stat：ranks 必須 sort（從正到負）並無 NA。亦避免用 padj 排——應用 LFC 或 limma t/DESeq2 stat。
Unsorted or NA ranks for GSEA: sort by descending stat and drop NAs. Don't rank by padj — use LFC or limma t/DESeq2 stat.
🚫
把上下調混在一起 ORA：建議分開 (up / down) 兩個 list 跑 ORA；GSEA 則保留方向資訊故無此問題。
Mixing up/down in ORA: split into up/down lists for ORA; GSEA preserves direction by design.
