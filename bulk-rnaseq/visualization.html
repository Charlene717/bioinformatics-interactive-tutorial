圖表說明

五張圖怎麼看

📍 PCA

前兩個主成分軸的點圖。同條件聚成一團 = 好；條件夾雜批次 = 需校正。

Scatter on first two PCs. Same-condition cluster = good; condition mixed with batch = needs correction.

🌋 Volcano

X = log2FC、Y = -log10(padj)。右上 = 顯著上調、左上 = 顯著下調、底部 = 不顯著。

X = log2FC, Y = -log10(padj). Upper right = significant up; upper left = significant down; bottom = NS.

🦋 MA-plot

X = log mean expression、Y = LFC。理想情境下，多數點應聚在 LFC = 0 附近，DEG 散落兩側。

X = log mean expression, Y = LFC. Ideally most points cluster around LFC = 0; DEGs scatter above/below.

🔥 Heatmap

取 vst/rlog 後選 top 50–100 DEGs；列 z-score (row scaling)，看樣本間表達模式。

vst/rlog values, top 50–100 DEGs, row z-score; reveals per-sample patterns.

程式碼

實作範例

vsd <- vst(dds, blind = FALSE)
plotPCA(vsd, intgroup = c("condition", "batch"))
# 自訂 ggplot
pcaData <- plotPCA(vsd, intgroup = "condition", returnData = TRUE)
ggplot(pcaData, aes(PC1, PC2, color = condition)) + geom_point(size = 3)
library(EnhancedVolcano)
EnhancedVolcano(res,
  lab        = rownames(res),
  x          = "log2FoldChange",
  y          = "padj",
  pCutoff    = 0.05,
  FCcutoff   = 1,
  labSize    = 3)
plotMA(res, ylim = c(-3, 3))
# shrinkage 後的 MA-plot 更乾淨
res_s <- lfcShrink(dds, coef = "condition_treated_vs_control", type = "apeglm")
plotMA(res_s)
library(pheatmap)
top <- head(order(res$padj), 50)
mat <- assay(vsd)[top, ]
mat <- mat - rowMeans(mat)   # row centering
pheatmap(mat,
  scale         = "row",
  cluster_cols  = TRUE,
  annotation_col = info[, "condition", drop = FALSE])
陷阱

常見錯誤

🚫
用 raw count 畫 PCA：variance 由 library size 主導，PCA 只反映 library size。請先 vst/rlog。
PCA on raw counts: variance is dominated by library size; the PCA only reflects depth. Always vst/rlog first.
🚫
Heatmap 沒做 row scaling：高表達基因永遠紅、低表達永遠藍，看不出模式。請用 scale = "row"。
Heatmap without row scaling: high expressors always red, lows always blue; no pattern visible. Use scale = "row".
🚫
Volcano 用 p-value 而非 padj：未校正多重比較，門檻線會嚴重低估假陽性。
Volcano on raw p-values: not multiple-test corrected; cutoff lines drastically understate false positives.
