STEP 3 / 7

表達量定量

從 BAM 或 reads 計算每個基因 / 轉錄本的表達量——這是進入統計分析前的最後一步資料整理。

Count reads per gene / transcript from BAM or directly from FASTQ — the last data-prep step before statistics.

核心概念

什麼是「表達量」

「表達量」(expression) 其實是個比較概念。在統計上，最原始、最有資訊量的表示是 raw read count——某基因有幾條 reads 被分配到它身上。所有合理的 DE 工具 (DESeq2、edgeR、limma-voom) 都從 raw count 開始建模。

各種「TPM / FPKM / CPM」都是把 count 換算成可以跨樣本／跨基因比較的「標準化單位」，但這些不是 DE 統計的最佳輸入。

"Expression" is fundamentally a comparison. Statistically, the rawest, most informative representation is a raw read count — how many reads were assigned to a gene. All proper DE tools (DESeq2, edgeR, limma-voom) model raw counts.

TPM / FPKM / CPM are normalized units intended for cross-sample / cross-gene comparison; they are not the right input for DE statistics.

單位公式何時使用
raw countN_gDE 統計
CPM10⁶ · N_g / Σ N跨樣本比較
FPKM / RPKM10⁹ · N_g / (L_g · ΣN)樣本內比較
TPM10⁶ · (N_g/L_g) / Σ (N/L)跨樣本比較
⚠️
常見誤用：用 TPM 直接做 t-test 來找 DE 是錯的——TPM 已喪失樣本間總量資訊，且未建模 mean-variance 關係。請永遠把 raw count 餵給 DESeq2/edgeR/voom。
Common pitfall: Running a t-test on TPMs to find DEGs is wrong — TPM loses total-library info and ignores the mean-variance relationship. Always feed raw counts to DESeq2 / edgeR / voom.
工具

兩條主流路徑

📊

路徑 A：BAM → featureCounts

先用 STAR/HISAT2 得到 BAM，再用 featureCounts（Subread 套件）或 HTSeq-count 在 GTF 註釋上計算每個基因 reads。

Get BAM from STAR/HISAT2, then count with featureCounts (Subread) or HTSeq-count against a GTF.

⚡

路徑 B：Salmon → tximport

Salmon / kallisto 直接給轉錄本層級表達量；tximport 把轉錄本估計值整合成基因層級的 counts（含 lengthScaledTPM 校正）。

Salmon / kallisto give transcript-level estimates; tximport aggregates to gene-level counts with proper length scaling.

決策

關鍵設定

🌳 featureCounts 必檢三項

-s：
Strandedness。0=非鏈特異、1=正鏈（如 dUTP 為 2=反鏈）。設錯會少算 50% 以上 counts！可用 infer_experiment.py 驗證。
-p：
Paired-end。雙端定序要加 -p --countReadPairs，否則 R1+R2 會被當成兩條獨立 reads（雙倍灌水）。
-t -g：
Feature 類型。預設 -t exon -g gene_id 統合 exons → gene 計數，符合大部分 DE 流程。
-s:
Strandedness. 0=unstranded, 1=forward, 2=reverse (e.g., dUTP). Wrong setting drops > 50% counts! Verify with infer_experiment.py.
-p:
Paired-end. Add -p --countReadPairs for PE; otherwise R1 + R2 count as two independent reads (double-counting).
-t -g:
Feature levels. Default -t exon -g gene_id aggregates exons → gene for standard DE flow.

程式碼

實作範例

# Paired-end, reverse-stranded (dUTP), gene-level count
featureCounts \
  -a gencode.v44.gtf \
  -F GTF -t exon -g gene_id \
  -p --countReadPairs \
  -s 2 -T 8 \
  -o counts.tsv \
  star_out/*.bam
htseq-count -f bam -r pos \
  -s reverse -t exon -i gene_id \
  -m union --nonunique=none \
  sample1.bam gencode.v44.gtf > sample1.counts.tsv
library(tximport)
library(GenomicFeatures)
# 1. 建立 tx2gene 對應表
txdb <- makeTxDbFromGFF("gencode.v44.gtf")
k    <- keys(txdb, keytype = "TXNAME")
tx2gene <- select(txdb, keys = k, columns = "GENEID", keytype = "TXNAME")
# 2. 整合多個 salmon 結果
files <- file.path("salmon_out", samples, "quant.sf")
names(files) <- samples
txi <- tximport(files, type = "salmon", tx2gene = tx2gene)
# txi$counts 可直接餵給 DESeqDataSetFromTximport()
