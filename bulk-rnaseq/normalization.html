STEP 4 / 7

標準化

校正樣本間的 library size 與組成偏差，讓不同樣本間的 counts 真正可比。

Correct for library-size and composition biases so counts from different samples become comparable.

為什麼要標準化

兩種樣本間的偏差

📦

Library size 偏差

A 樣本 30M reads、B 樣本 50M reads，即使表達一致，B 的 raw counts 自然較高。

If A has 30 M reads and B has 50 M, raw counts in B will be higher even with identical biology.

⚖️

Composition 偏差

某幾個基因暴衝（如肝臟 albumin），會「吃掉」其他基因的 reads share；單純除 library size 無法校正。

If a few genes dominate (e.g., liver albumin), they steal share from others — dividing by library size alone cannot fix it.

💡
核心想法：多數基因不會有差異表達，因此這些「house-keeping」基因在 case/control 中應該大致一樣多——這個假設是 TMM、DESeq2 size factor 的基礎。
Core assumption: Most genes are not differentially expressed; "housekeeping" genes should be roughly equal across case/control. This underpins TMM and DESeq2 size factors.

Composition 偏差如何欺騙簡單除法

樣本 B 有一個「巨星基因」暴衝（slider 控制），會偷掉其他基因的 read share。觀察 raw、CPM、TMM-normalized 中其他基因的相對量：

Sample B harbors a "superstar gene" stealing read share from others. See how raw / CPM / TMM-normalized values for the other genes compare:

⚠️
當巨星基因強到一定程度，CPM 會錯誤地把其他基因（其實沒變化）顯示為「下降」。TMM/RLE 透過取中位數比例可避開這個假象。
A strong-enough superstar can make all other (unchanged) genes look "down-regulated" under CPM. TMM/RLE avoid this by anchoring on median ratios.
程式碼

實作範例

library(edgeR)
y <- DGEList(counts = mat, group = grp)
y <- calcNormFactors(y, method = "TMM")
# 標準化後的 CPM（log）
logcpm <- cpm(y, log = TRUE, prior.count = 1)
library(DESeq2)
dds <- DESeqDataSetFromMatrix(countData = mat, colData = info, design = ~condition)
dds <- estimateSizeFactors(dds)
sizeFactors(dds)   # size factor / 樣本
norm_counts <- counts(dds, normalized = TRUE)
# vst (推薦)：快、變異穩定；適合 PCA、heatmap、ML
vsd <- vst(dds, blind = FALSE)
plotPCA(vsd, intgroup = "condition")

# rlog：小樣本(<30)時較穩，但慢
rld <- rlog(dds, blind = FALSE)

該選哪個？

🌳 按用途挑方法

DE 統計：
不用做額外標準化——直接 raw counts 餵 DESeq2/edgeR，工具內建 size factor。
PCA / heatmap：
vst（推薦）或 rlog（n < 30）。
基因表達比較表：
取 TPM（log 轉換後）作為跨樣本展示。
機器學習特徵：
vst 後再做 z-score；勿用 raw counts 直接訓練。
DE statistics:
No extra step — feed raw counts to DESeq2/edgeR; size factors are computed internally.
PCA / heatmap:
vst (recommended) or rlog (small n < 30).
Expression tables:
log-TPM for cross-sample display.
ML features:
vst then z-score; never train on raw counts directly.
